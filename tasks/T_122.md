# T_122: Guarantee the ngrok tunnel is killed on app exit

## Status

Done

## Summary

Harden ngrok tunnel teardown so the spawned `ngrok` CLI process cannot be left running after the app
closes. Previously `app.on("will-quit")` called `void ngrokTunnel.stopTunnel()` — fire-and-forget,
and the actual `child.kill()` ran through an async `runExclusive` chain, so it was not guaranteed to
execute before the process terminated. Now the kill is synchronous on `will-quit`, with a
`process.on("exit")` fallback for graceful exit paths that bypass `will-quit`. (A true `SIGKILL`/crash
still cannot run any cleanup — nothing can — but the realistic graceful-exit gaps are closed.)

## Implementation Notes

- `electron/ngrok-tunnel.cjs`: add and export `killTunnelSync()` — a synchronous, idempotent teardown
  that calls the existing `killChild()` (`child.kill()`) and resets state directly, bypassing the
  async `runExclusive` queue. Safe to call repeatedly and when no tunnel is active.
- `electron/main.cjs`:
  - In `app.on("will-quit")`, replace `void ngrokTunnel.stopTunnel()` with `ngrokTunnel.killTunnelSync()`
    so the agent is killed before the process exits.
  - Add `process.on("exit", () => ngrokTunnel.killTunnelSync())` as a last-resort synchronous fallback
    for graceful exits that do not go through `will-quit`.
- The agent is spawned non-detached (`spawn(command, args, { windowsHide: true })`), so a kill
  terminates it; no job-object/process-tree handling is needed for the single ngrok process.

## Known limitation

A hard kill of the Electron main process (Task Manager "End task", `kill -9`, power loss, or a native
crash) runs no JavaScript, so neither handler fires and the non-detached child can be orphaned. This
is inherent to those termination modes and is out of scope.

## Verification

- `npm run build` (`tsc --noEmit` + vite) and `npm run test` pass (274 tests).
- `electron/ngrokTunnel.test.ts` (new): asserts `killTunnelSync()` does not throw and is idempotent
  when no tunnel is active (the contract the `process.on("exit")` handler relies on, since it runs on
  every exit including when ngrok was never started), leaving `getTunnelState()` disconnected.
- `node --check` on `electron/ngrok-tunnel.cjs` and `electron/main.cjs`.
- Manual (desktop): start the ngrok tunnel, quit the app (Cmd/Ctrl-Q and window close), and confirm
  with `Get-Process ngrok` / `pgrep ngrok` that no `ngrok` process remains.
