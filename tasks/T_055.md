# T_055: Focus the initial empty editor on launch

**Task Status:** Completed

**Task Description:** Put keyboard focus in the editor when Nexus launches to an empty untitled document so the caret is visible and the user can type immediately.

## Context

Nexus starts each normal app window with a blank untitled Markdown buffer. That first screen should behave like a writing surface: once startup confirms no file is being opened into the window, focus should land in the editor rather than requiring a click.

## Acceptance Criteria

1. A normal launch with an empty untitled document focuses the editor automatically.
2. The caret appears at the start of the empty document.
3. Windows launched with an OS-provided file path do not steal focus before or after the file loads.
4. The focus behavior runs once at startup and does not keep refocusing after later edits or state changes.

## Implementation Notes

- Reused MDXEditor's `focus` method with `defaultSelection: "rootStart"` and `preventScroll: true`.
- Waited for the initial-open-file IPC result before focusing, so OS-opened files are not treated as blank startup documents.
- Added refs that guard both initial file handling and initial empty-editor focus as one-shot startup actions.

## Verification

Run `npm run build`, `npm run test`, and `git diff --check`. Launch Nexus normally and confirm the caret appears in the empty editor without clicking.
