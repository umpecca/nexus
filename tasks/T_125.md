# T_125: Add a context-rich MCP find tool

## Status

Done

## Summary

`nexus_search_document` (T_117) already locates text, but each match returns only the single matching
line — no surrounding context and no indication of where in the document structure it sits, so an LLM
usually needs a follow-up read to understand a hit. Add a comprehension-oriented `nexus_find` read
tool: same literal/regex search, but matches are grouped by line and each carries a few surrounding
context lines and the heading the line falls under, so a client can understand what it found and where
in one call. `nexus_search_document` stays as the lightweight per-occurrence positions/counts tool;
the two cross-reference each other in their descriptions so an LLM picks the right one.

## Implementation Notes

- `electron/mcpDocumentTools.cjs`: add `findInDocument(markdown, options)`. Reuses `escapeRegExp`,
  `truncatePreview`, `parseHeadings`, and `toPublicHeading`. Groups occurrences by line into entries
  `{ line, columns, matchCount, heading, contextStartLine, context }`, where `heading` is the nearest
  heading at or above the line (`findEnclosingHeading`, null when the line precedes all headings) and
  `context` is the `contextLines` lines on each side joined and per-line truncated. Returns
  `{ query, isRegex, caseSensitive, total, matchingLines, truncated, matches }`. Caps matching lines
  at `maxResults` (default 50, max 500) while still counting accurate totals, bounds `contextLines`
  (default 2, max 10), guards zero-width regex matches, and throws on an empty query or invalid regex.
- `electron/main.cjs`: add the `mcpFind` host method (mirrors `mcpSearchDocument`) and register it as
  `find` on `mcpServer.setHost`.
- `electron/mcp-server.cjs`: add the `nexus_find` tool definition to `READ_ONLY_TOOLS` and a dispatch
  branch (mirrors `nexus_search_document`: non-empty-query check, try/catch around the host call for
  invalid-regex/empty-query errors, no-window handling). Added a cross-reference sentence to the
  `nexus_search_document` description.
