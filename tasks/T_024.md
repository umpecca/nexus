# T_024: Start each launch with a blank untitled document

**Status:** Done

**Task Description:** Change Nexus startup behavior so every app launch begins with a clean blank untitled document instead of restoring previous local draft content.

## Acceptance Criteria

1. Launching the app starts with an empty editor.
2. Launching the app starts with no current file path.
3. The initial window title is the clean untitled title state.
4. The initial buffer is not dirty until the user edits it.
5. File/Open, File/Save, File/Save As, File/New, and close dirty-buffer prompts continue to work after the startup change.

## Implementation Notes

- Changed the default Markdown document to an empty string.
- Changed `App` startup to initialize from `createDefaultDraft()` rather than `loadDraft()`.
- Kept the existing current-state save helper available, but the renderer no longer restores prior local storage content during app startup.
- Updated the draft helper test to expect a blank default draft.

## Verification

- Run `npm run build`.
- Run `npm run test`.
