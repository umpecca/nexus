# T_115: Outline scroll-spy and source-mode outline support

## Status

Done

## Summary

Deliver the two outline items previously parked under PRODUCT.md "Future Iterations": highlight the
active section in the outline based on the editor's scroll position, and support the outline while
editing in source mode. As the user scrolls (in rich-text or source mode) the outline now highlights
the entry for the section under the top of the viewport, and the sidebar — selection-scroll and
scroll-spy — works in source mode as well as rich-text, hiding only in diff mode where the full width
is wanted for side-by-side review. The heading-extraction infrastructure already existed; this is the
wiring from scroll position and editor mode to the active entry, plus precise source-mode positions.

## Implementation Notes

- `src/lib/outline.ts`:
  - Add a zero-based `line` (source line index) to `OutlineHeading` and populate it in
    `extractOutline` so source mode can map an entry to its CodeMirror line.
  - Add the pure `getActiveHeadingIndex(headingTops, viewport, activationOffset)` scroll-spy helper:
    the active heading is the last whose top is at or above an activation line just below the viewport
    top, with the final heading winning at the very bottom and the first heading held above the first
    heading. Unresolved tops are passed as `Infinity` so they never win.
- `src/lib/outline.test.ts`: update the `toEqual` expectations for the new `line` field and add
  cases for line tracking and for `getActiveHeadingIndex` (above-first, mid-document, on-the-line,
  bottom edge, and `Infinity` handling). 88 tests pass overall.
- `src/components/editor/OutlineSidebar.tsx`: accept an `activeIndex`, mark the matching entry with
  `nexus-outline-item-active` + `aria-current`, change `onSelect` to pass the whole `OutlineHeading`
  (so the app shell has the line for source mode), and nudge only the outline list (never the page or
  editor) to keep the active entry visible as it changes.
- `src/App.tsx`:
  - Module helpers: `getSourceEditorView` reaches MDXEditor's CodeMirror `EditorView` through the
    content DOM node's `cmTile.view` (no direct `@codemirror/view` dependency), guarded so any
    internals change degrades gracefully; `getSourceHeadingTop` uses `view.lineBlockAt(...).top` (the
    height model, accurate for virtualized off-screen lines); `getOutlineHeadingMetrics` returns each
    heading's content-top offset for the current mode (rendered `h1..h6` geometry in rich-text).
  - Extend `showOutlineSidebar` from rich-text-only to `editorViewMode !== "diff"`.
  - Extend `scrollOutlineHeadingIntoView` to take an `OutlineHeading` and scroll the CodeMirror
    scroller to the heading line in source mode (rich-text path unchanged).
  - Add a scroll-spy effect (deps: outline visibility and editor mode) that binds scroll listeners to
    the active editor scroll elements, recomputes the active heading on scroll/resize/DOM-mutation
    (throttled to an animation frame, latest headings read through a ref), and rebinds when a mode
    switch mounts a new scroller. Pass `activeIndex` to `OutlineSidebar`.
- `src/styles.css`: add `.nexus-outline-item-active` (accent text, inset accent left bar, tinted
  background, placed after the level rules so it overrides muted deep levels) and extend the
  outline room-reservation padding to `.mdxeditor-source-editor` so the overlay never covers source
  text.
- Docs: PRODUCT.md (capability, requirements, primary workflow; remove the two implemented Future
  Iterations), ARCHITECTURE.md (OutlineSidebar/`outline.ts` descriptions, flow items 53–54, the
  Document Outline Sidebar workflow), and this changelog entry.

## Verification

- `npm run build` (`tsc --noEmit` + vite) and `npm run test` (88 tests) pass.
- Browser preview (rich-text and source mode, with a 10-section scrollable document): the active
  outline entry tracks the scroll position monotonically and snaps to the last heading at the bottom;
  clicking an entry scrolls to that heading (verified to land on the exact CodeMirror line offset in
  source mode); the source-mode outline reserves its width (no overlap) and the active entry shows the
  accent color, left bar, and `aria-current`.
  - Note: the preview tab runs hidden, so `requestAnimationFrame` (and `requestIdleCallback`) are
    paused there; the rAF-throttled scroll-spy recompute was verified by shimming rAF to a timer,
    which is what a visible window does. In the real Electron window rAF fires normally.
- Manual test (desktop): scroll a long document in rich-text and confirm the highlighted heading
  follows; repeat in source mode; switch to diff mode and confirm the outline hides; toggle the
  outline off/on and confirm the highlight resumes.
