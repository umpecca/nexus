# T_123: Add OAuth 2.1 support to the embedded MCP server (ChatGPT connectors)

## Status

Done

## Summary

ChatGPT custom connectors (and Claude.ai remote connectors) authenticate to remote MCP servers
through the MCP authorization spec — OAuth 2.1 with discovery metadata, dynamic client registration,
and the authorization-code grant with PKCE. Nexus only accepted a static bearer token, so connecting
from ChatGPT failed with "the server does not implement OAuth." Add a minimal OAuth surface to the
embedded server so those clients can connect, without changing the security model:

- The flow ends by issuing the server's **existing static bearer token** as the access token, after
  the user approves a consent page in their browser. `/mcp` validation is unchanged, OAuth-issued
  access survives app restarts, and "Regenerate token" in settings remains the single kill switch
  that revokes every client.
- The OAuth surface is active only in bearer-token mode (in no-auth mode nothing returns 401 and
  there is no token to issue, so the endpoints 404).
- The write-confirmation dialog / auto-approve setting continue to gate writes exactly as before.

## Implementation Notes

- `electron/mcpOauth.cjs` (new): protocol logic, no Electron or HTTP dependencies.
  - RFC 9728 protected-resource metadata (`resource: {origin}/mcp`) and RFC 8414 authorization-server
    metadata (`/authorize`, `/token`, `/register`, S256-only, public clients).
  - RFC 7591 dynamic client registration: validates redirect URIs (https anywhere, plain http only on
    loopback hosts), issues a `nexus-…` client id, caps the store at 100, and persists registrations
    (id/name/redirect URIs — no secrets) best-effort to a JSON file so reconnects survive restarts.
  - `beginAuthorization` validates the request per RFC 6749 §4.1.2.1 (unknown client/redirect → an
    in-place error page, never a redirect; missing PKCE → `invalid_request` redirect) and stores a
    pending request (10-min TTL) with a consent nonce.
  - `decideAuthorization` requires the nonce (CSRF: a cross-origin page cannot read the consent HTML),
    deletes the pending request either way, and redirects with a single-use code (5-min TTL) or
    `access_denied`.
  - `exchangeAuthorizationCode` enforces single-use codes, client/redirect match, and S256 PKCE
    (timing-safe compares), then issues the supplied bearer token; `expires_in` is omitted.
- `electron/mcp-server.cjs`:
  - 401 responses on `/mcp` and `/health` now carry the RFC 9728 challenge
    `WWW-Authenticate: Bearer resource_metadata="{origin}/.well-known/oauth-protected-resource"`.
  - Routes (bearer mode only): both well-known metadata paths plus the `/mcp`-suffixed variants and
    `/.well-known/openid-configuration`, `POST /register`, `GET /authorize` (renders a styled consent
    page showing the client name and redirect host, with Approve/Deny), `POST /authorize/decision`,
    and `POST /token`.
  - `getRequestOrigin` builds metadata/challenge origins from the Host header plus ngrok's
    `X-Forwarded-Proto`, so discovery URLs are correct both on loopback and through the tunnel.
  - Consent/error pages send `X-Frame-Options: DENY`, `frame-ancestors 'none'`, and
    `Referrer-Policy: no-referrer`; `sendJson`/`sendHtml` accept extra headers.
  - `configure()` passes `oauthClientStorePath` through to `mcpOauth.configure`; disable/stop clears
    the volatile pending-request/code state (persisted registrations survive).
- `electron/main.cjs`: passes `oauthClientStorePath: {userData}/mcp-oauth-clients.json` (same pattern
  as `recent-files.json`).
- `src/components/settings/SettingsDialog.tsx`: the MCP help text now mentions OAuth (and reflects
  that writes are confirmed unless auto-approve is on).

## Notes

- ChatGPT requires an https URL, so connecting uses the ngrok public URL; on the free ngrok tier the
  browser consent page may show ngrok's interstitial first ("Visit Site"), then the consent renders.
- Handing the static token to OAuth clients is a deliberate single-user trade-off: it keeps the
  server stateless for auth and restart-safe; revoking = Regenerate token.
