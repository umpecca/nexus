# T_132: Render OpenAPI references in print and PDF exports

## Status

Done

## Summary

Replace raw `yaml openapi` fences in printable and exported document HTML with a static OpenAPI API
reference matching the rich-text preview, so printed and PDF architecture documents remain visual
and useful.

## Acceptance Criteria

- HTML used for printing and PDF export renders valid `yaml openapi` fences as static API reference
  cards rather than YAML code.
- The export shows API title/version/summary, servers, tagged operations, method/path details,
  parameters, security, request bodies, examples, responses, and resolved schema properties.
- Exported operations are expanded; no Hide reference or Edit visually controls are present.
- Print styling uses method colors, readable borders, page-break-safe operation cards, and works in
  both light HTML export and PDF printing.
- Ordinary YAML fences and malformed OpenAPI fences remain syntax-preserving code blocks rather than
  breaking export.
- The implementation reuses the existing OpenAPI reference model so WYSIWYG and export content stay
  aligned.
- Tests cover fence detection, safe escaping, valid static reference rendering, normal YAML fallback,
  malformed OpenAPI fallback, and omission of interactive controls.
- PRODUCT, ARCHITECTURE, CHANGELOG, and this task record are updated.

## Implementation Notes

- Transform only fenced blocks carrying the `openapi` metadata token.
- Generate self-contained semantic HTML/CSS in the export pipeline; do not load Swagger UI or remote
  assets in the hidden PDF window.
