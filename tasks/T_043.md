# T_043: Use Nexus PNG as the Electron app icon

**Task Description:** Use the local `nexus.png` asset as the application icon for Nexus windows.

**Task Status:** Complete

**Task Status Details:** Added main-process icon path resolution and passed the asset into Electron BrowserWindow creation, with macOS Dock icon support when available.

## Context

Nexus should identify itself like a desktop application instead of using Electron's default icon. The user added `nexus.png` to the project root and asked for it to become the app icon.

## Acceptance Criteria

1. Electron windows use the local `nexus.png` file as their icon when the asset exists.
2. The macOS Dock uses the same PNG when running on macOS.
3. The app still launches if the icon file is missing.
4. The change does not alter renderer behavior or document state.

## Implementation Notes

- Added a root-relative `appIconPath` in `electron/main.cjs`.
- Added `getAppIconPath()` to avoid failing startup when the file is absent.
- Passed the icon path to `new BrowserWindow()`.
- Set `app.dock` icon on macOS after Electron is ready.

## Verification

Run `node --check electron/main.cjs`, `node --check electron/preload.cjs`, `npm run test`, and `npm run build`. Manually launch Electron and confirm the app/window icon uses `nexus.png`.
