# T_018-Traditional File New Dirty Buffer Handling

**Task ID:** T_018

**Task Description:** Make File/New behave like a traditional desktop application action by protecting unsaved changes before clearing the current document.

**Status:** CLOSED

## Acceptance Criteria

1. File/New clears the editor content instead of loading the default starter document.
2. File/New clears the current file path after creating the new blank document.
3. If the current buffer has unsaved changes, File/New prompts the user to Save, Don't Save, or Cancel.
4. Choosing Save saves through the existing Save or Save As flow before creating the new blank document.
5. Choosing Don't Save creates the new blank document without saving.
6. Choosing Cancel leaves the current buffer and file path unchanged.

## Implementation Notes

- Added an Electron message-box IPC handler for the unsaved-change prompt.
- Exposed `confirmSaveChanges` through preload and TypeScript renderer types.
- Tracked the last saved/opened Markdown buffer in the renderer.
- Updated the File/New menu action handler to branch through save, discard, or cancel before clearing the editor.

## Verification

Run `npm run build` and `npm run test`. In Electron, edit the current document and choose File/New; verify Save, Don't Save, and Cancel each follow traditional desktop behavior.
