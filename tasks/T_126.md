# T_126: Preserve WYSIWYG insertion point for native diagrams

## Status

Done

## Summary

In a large document, inserting a drawio diagram from the WYSIWYG toolbar can lose the caret while
the separate diagram editor is open. When the async editor returns, the current implementation sees
no live Lexical range selection and deliberately selects the end of the document, so the new diagram
is inserted there instead of at the requested location. Preserve the intended editor and caret before
opening drawio (and the equivalent isoflow workflow), then restore that selection when inserting the
returned diagram.

## Acceptance Criteria

- Starting a drawio insertion at a valid WYSIWYG caret inserts the saved diagram at that caret even
  after focus and the live DOM/Lexical selection are lost during the native editor round trip.
- The same shared native-diagram path behaves consistently for isoflow.
- Nested editors, such as callouts and table cells, continue to receive and persist the insertion.
- If the captured selection is no longer valid, insertion fails soft using the existing end-of-target
  fallback instead of throwing or corrupting the document.
- Automated tests cover selection preservation and fallback behavior, and the project build/tests pass.

## Implementation Notes

- `src/components/editor/InsertDiagram.tsx` captures a cloned Lexical range selection synchronously
  from the selected root or nested editor before awaiting the native drawio/isoflow bridge.
- When the SVG returns, the insertion update verifies that both saved selection nodes still exist,
  restores a fresh clone with `$setSelection`, and inserts at that range. A missing/stale selection
  retains the existing target-editor-end fallback.
- `src/components/editor/InsertDiagram.test.ts` exercises the focus-loss path by clearing the live
  selection before insertion and covers the stale-node fallback after removing the captured target.
