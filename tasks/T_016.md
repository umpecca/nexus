# T_016-Fix Shadcn Toolbar Wrapping And Dropdown Stacking

**Task ID:** T_016

**Task Description:** Fix the shadcn-styled MDXEditor toolbar so wrapped controls remain orderly and dropdown or popover panels are not covered by the sticky toolbar.

**Status:** CLOSED

## Acceptance Criteria

1. Toolbar controls wrap as coherent command groups instead of relying on broad internal MDXEditor flex selectors.
2. Rich-text controls and the rich/diff/source toggle share the toolbar predictably across narrower widths.
3. Dropdown, select, and popover panels opened from toolbar controls appear above the sticky toolbar.
4. The toolbar remains sticky and does not overlap editable document content.

## Implementation Notes

- Added explicit wrapper classes for rich toolbar controls and the command cluster.
- Removed the broad selector that altered every inline `display: flex` element inside the toolbar.
- Tightened toolbar spacing and button sizing so the default desktop toolbar does not orphan the final controls onto a second row.
- Overrode MDXEditor dropdown, select, and popover stacking so floating panels render above the sticky toolbar.

## Verification

Run `npm run build` and `npm run test`. Open the app and verify the toolbar wraps cleanly and the block type/admonition/dropdown panels are fully visible above the toolbar.
