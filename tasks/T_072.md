# T_072: Add a saved light, dark, and system-following app theme

## Status

Done

## Summary

Add a Settings preference that lets users choose System, Light, or Dark for the Nexus app theme. The preference is saved per OS profile, System follows the desktop color-scheme setting, and HTML/PDF export rendering remains light and predictable.

## Implementation Notes

- Added a `themePreference` field to profile-scoped user settings with validation and a default of `system`.
- Added a Theme select to the Settings dialog.
- Resolved the active app theme in `App.tsx`, including live `prefers-color-scheme` updates while System is selected.
- Applied the resolved theme through `data-theme` and `color-scheme`.
- Converted app chrome, dialogs, toolbar groups, source/diff surfaces, settings controls, and editor utility blocks to theme-aware CSS tokens.
- Kept export styling unchanged so exported documents do not inherit the UI dark theme.

## Verification

- Unit tests cover default, valid, invalid, and saved theme preference behavior.
