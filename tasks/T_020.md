# T_020: Put Nexus first in the title and mark unsaved documents

**Status:** Done

**Task Description:** Update the native application title so it follows a traditional desktop format with the app name first and a dirty-document marker when the buffer has unsaved changes.

## Acceptance Criteria

1. An untitled clean document shows `Nexus - Untitled` in the application title.
2. An untitled modified document shows `Nexus - *Untitled` in the application title.
3. A clean opened or saved document shows `Nexus - filename.md (full path)` in the application title.
4. A modified opened or saved document shows `Nexus - *filename.md (full path)` in the application title.
5. The dirty marker clears after a successful Save or Save As and returns when the user edits again.

## Implementation Notes

- Updated the title formatter in `src/App.tsx` to put the app name first.
- Promoted the last saved/opened Markdown baseline into React state so title dirty state rerenders immediately after edits and saves.
- Kept File/New, File/Open, File/Save, and File/Save As aligned with the same dirty baseline.
