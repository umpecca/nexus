# T_057: Add adjustable visual editor and PDF margins

**Task Status:** Completed

**Task Description:** Add per-side page margin controls so the visual paper editor and PDF export use the same printable content boundaries.

## Context

Nexus now supports Letter and A4 paper sizing, but the initial print-layout experiment used fixed one-inch margins. Users who export and print often need tighter or wider margins to match real paperwork requirements and prevent visual sizing surprises between editing and PDF output.

## Acceptance Criteria

1. Settings allows adjusting top, right, bottom, and left margins.
2. Margin settings are stored per OS profile with the existing editor settings.
3. Rich-text editing applies the selected margins to the paper-width page surface.
4. PDF export uses the selected margins.
5. Existing font settings, paper-size settings, HTML export, source mode, diff mode, image preview, Mermaid export, and admonition export continue to work.

## Implementation Notes

- Extended the local settings model with `pageMargins`, defaulting each side to one inch.
- Limited margin controls to 0.25 through 2 inches in 0.25-inch steps.
- Added side-specific paper margin CSS variables for the rich-text editor surface.
- Passed the selected margins through preload IPC and validated them in the Electron main process before calling `printToPDF`.
- Left true pagination out of scope.

## Verification

Run `npm run test`, `npm run build`, and `git diff --check`. Manually change each margin in Settings, confirm the rich-text page updates, then export PDFs with default, narrow, and wide margins.
