# T_096: Add the embedded MCP HTTP server with read-only document tools

## Status

Done

## Summary

Stand up the embedded Model Context Protocol HTTP server inside the Electron main process. It listens on `127.0.0.1` only, authenticates with the per-profile bearer token configured in T_095, and exposes the two read-only tools `nexus_list_windows` and `nexus_get_document` over Streamable HTTP JSON-RPC. No write tools yet.

## Implementation Notes

- Add `electron/mcp-server.cjs` that exports `{ configure, stop }`. `configure({ enabled, port, bearerToken })` starts, stops, or reconfigures a single `http.Server` instance. Bind to `127.0.0.1` only. Restart the listener when the port changes; just rotate the in-memory token without restarting when only the token changes.
- Implement a Streamable HTTP transport at `POST /mcp`: parse JSON-RPC bodies, validate `Authorization: Bearer {token}` (constant-time compare), reject with 401 on mismatch and 400 on malformed JSON. Respond `Content-Type: application/json`. Respond to `GET /mcp` with 405 for now (SSE streaming is unused in v1).
- Implement the JSON-RPC methods needed by current MCP clients: `initialize` (return server info `{ name: "nexus-mcp", version: <package.version> }` and `capabilities: { tools: {} }`), `notifications/initialized` (no response), `tools/list` (return tool definitions), `tools/call` (dispatch to tool implementations and return `{ content: [{ type: "text", text: ... }] }`). Return `-32601` for unknown methods.
- Add a per-window MCP record map keyed by a renderer-supplied stable window ID. The record holds `{ windowId, webContentsId, title, filePath, dirty, markdown, focused }`. Add IPC channels:
  - `mcp:register-window` (renderer -> main) — main creates or updates the record for the calling renderer.
  - `mcp:update-window-state` (renderer -> main) — renderer pushes latest `{ markdown, filePath, dirty, title }` so the read tools can serve them without round-tripping.
  - `mcp:unregister-window` (renderer -> main, on unload) — main drops the record.
  - Track focused window via `browser-window-focus` and `browser-window-blur` Electron events.
- Implement the two tools in `electron/mcp-server.cjs`:
  - `nexus_list_windows` (no args): returns the list of registered windows with `{ windowId, title, filePath, dirty, focused }`.
  - `nexus_get_document` (args: optional `windowId` string): returns `{ windowId, title, filePath, dirty, markdown }` for the focused window when `windowId` is omitted, or for the specified window. Return a JSON-RPC error result if no window matches.
- Tool responses use MCP's `content` array shape with a single `type: "text"` entry containing JSON-serialized payload. Mark `isError` true on lookup failures.
- Extend `App.tsx` to register the window on mount (using a per-session random window ID generated once), push state on every relevant document state change (markdown / file path / dirty / title), and unregister on unmount/`beforeunload`.
- Extend the `mcp:configure` handler from T_095 so it forwards to `mcpServer.configure(...)`. Return `{ ok: false, error: "EADDRINUSE" | "EACCES" | ... }` shapes for bind failures so the settings dialog can later surface them.
- Defer the `nexus_replace_document` tool to T_097; declare a stub that returns a JSON-RPC error explaining the tool is not yet wired so any client that tries to call it during the gap behaves predictably.

## Verification

- Run `node --check electron/main.cjs`.
- Run `node --check electron/preload.cjs`.
- Run `node --check electron/mcp-server.cjs`.
- Run `npm run build`.
- Run `npm run test`.
- Enable MCP from the settings dialog, then from a separate terminal:
  - `curl -X POST -H "Authorization: Bearer <token>" -H "Content-Type: application/json" -d '{"jsonrpc":"2.0","id":1,"method":"initialize","params":{"protocolVersion":"2024-11-05","clientInfo":{"name":"curl","version":"0"},"capabilities":{}}}' http://127.0.0.1:39125/mcp` returns the server info.
  - `tools/list` lists `nexus_list_windows`, `nexus_get_document`, and the stub `nexus_replace_document`.
  - `tools/call` for `nexus_get_document` returns the currently focused document's markdown.
  - A request without an `Authorization` header returns 401.
