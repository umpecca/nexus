# T_058: Add base font size settings for editing and export

**Task Status:** Completed

**Task Description:** Add a profile-scoped base font size preference that controls the visual editor and rendered HTML/PDF exports.

## Context

Nexus now mirrors paper size and margins between editing and PDF export, but the exported document still used a fixed 16px base font size. Users preparing print-heavy documents need base text sizing to be adjustable and predictable across the editor, HTML export, and PDF export.

## Acceptance Criteria

1. Settings allows adjusting the base font size.
2. The base font size is stored per OS profile with the existing editor settings.
3. Rich-text and source editing apply the selected base font size.
4. HTML export uses the selected base font size.
5. PDF export uses the selected base font size.
6. Existing font family, paper size, margin, image preview, Mermaid export, admonition export, and frontmatter PDF handling continue to work.

## Implementation Notes

- Extended the local settings model with `fontSizePixels`, defaulting to 16.
- Limited font size controls to 12 through 24 pixels in one-pixel steps.
- Added an editor font-size CSS variable consumed by rich-text and source editing surfaces.
- Passed the selected font size through preload IPC for both HTML and PDF export.
- Validated export font size in the Electron main process before writing export CSS.

## Verification

Run `npm run test`, `npm run build`, and `git diff --check`. Manually adjust the base font size in Settings, confirm the editor changes, then export HTML and PDF to confirm rendered text follows the selected size.
