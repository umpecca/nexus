# T_052: Add a built-in feature demo document

**Task Status:** Completed

**Task Description:** Add a native menu action that loads a built-in Markdown document showcasing the editor and export features Nexus supports.

## Context

Nexus needs a simple repeatable demo and smoke-test document so toolbar changes, rich text rendering, source mode, and HTML/PDF export can be checked without manually recreating feature coverage each time.

## Acceptance Criteria

1. The File menu includes a `Load Demo Document` action.
2. Loading the demo document prompts before replacing unsaved edits.
3. The demo loads as a clean untitled document and does not point at a real file path.
4. The document showcases frontmatter, formatted text, links, base64 images, tables, task lists, thematic breaks, Mermaid diagrams, runnable JavaScript, standard code blocks, and admonitions.
5. Existing New, Open, Save, Save As, export, and close flows continue to work.

## Implementation Notes

- Added a bundled renderer-side demo Markdown document.
- Added a `loadDemo` menu action from the Electron File menu.
- Extended the renderer menu-action type and handler.
- Reused the existing dirty-buffer confirmation flow before replacing the current editor state.
- Loaded the demo through the existing programmatic document replacement path so stale editor change events do not mark it dirty.

## Verification

Run `npm run build`, `npm run test`, and `git diff --check`. Manually choose File/Load Demo Document, confirm the feature showcase appears, and export it to HTML/PDF as a smoke test.
