# T_022: Prompt before closing with unsaved changes

**Status:** Done

**Task Description:** Make every application close path protect dirty editor content with the same Save, Don't Save, or Cancel prompt used by File/New.

## Acceptance Criteria

1. Closing a clean document exits without prompting.
2. Closing a dirty document shows the native Save, Don't Save, or Cancel prompt.
3. Choosing Save writes the current buffer, then closes the app when saving succeeds.
4. Choosing Don't Save closes the app without writing the dirty buffer.
5. Choosing Cancel leaves the app open and preserves the current buffer.
6. File/Exit, the window close button, and keyboard/window-manager close attempts all use the same guard.

## Implementation Notes

- Added a main-process close guard that pauses `BrowserWindow` close events and sends a close request to the renderer.
- Exposed `onCloseRequest` and `resolveCloseRequest` through preload.
- Reused the renderer dirty-buffer confirmation flow for both File/New and close requests.
- Changed File/Exit to request a guarded window close instead of calling `app.quit()` directly.
