# T_037: Prompt when opened files change outside Nexus

**Task Description:** Watch each opened document file and prompt the user when that file changes, moves, or is deleted outside Nexus.

**Task Status:** Complete

**Task Status Details:** Added a per-window Electron file watcher, preload watch/read APIs, renderer reload/conflict prompts, and document lifecycle hooks that start and stop watching as the current file path changes.

## Context

Nexus supports multiple editor windows and local file open/save workflows. Users can also edit the same Markdown file in another app or another Nexus window, so each window should detect external disk changes without confusing them with its own Save operation.

## Acceptance Criteria

1. Opened files are watched on macOS and Windows.
2. Clean editor buffers prompt the user to reload when the disk file changes externally.
3. Dirty editor buffers show a conflict prompt with Reload, Keep Editing, and Save As choices.
4. Saves from the same window do not trigger that window's reload prompt.
5. Saves from another Nexus window editing the same file still trigger an external-change prompt.
6. Moved or deleted watched files leave the editor buffer open and notify the user.

## Implementation Notes

- Added debounced `fs.watch` handles in the Electron main process keyed by renderer `webContents.id`.
- Added preload APIs for watching, unwatching, reading watched files, and subscribing to external change events.
- Added a shadcn-styled `FileChangedDialog` for reload, conflict, and missing-file prompts.
- Reused the existing editor dirty check, Save As flow, and document load function.

## Verification

Run `npm run test` and `npm run build`. Manually open a Markdown file, edit it outside Nexus, and verify reload/conflict/missing-file prompts across clean, dirty, and multi-window cases.
