# T_110: Add an optional custom ngrok executable path

## Status

Done

## Summary

Add a per-OS-profile option to point the MCP ngrok tunnel at an explicit ngrok executable instead of resolving `ngrok` from `PATH`. A checkbox in the MCP server settings toggles between PATH auto-detection (default) and a text field holding the path. This covers non-standard install locations, the Windows `.cmd`/`.bat` shim case, and selecting a specific ngrok build. When the checkbox is on but the path is empty, the tunnel falls back to PATH auto-detection so there is no half-configured error state.

## Implementation Notes

- Settings (`src/lib/settings.ts`):
  - Add `ngrokUseCustomPath: boolean` (default `false`) and `ngrokPath: string` (default `""`) to `McpServerSettings`.
  - Update `createDefaultMcpServerSettings` and `sanitizeMcpServerSettings` (coerce the boolean to a default `false` and the path to a trimmed string).
  - Update `src/lib/settings.test.ts`: include the two new fields in the full-object `createDefaultSettings`, font-only load, and save assertions, and extend the dedicated ngrok load/sanitize tests.
- Tunnel module (`electron/ngrok-tunnel.cjs`):
  - `ensureTunnel({ port, domain, command })` now accepts a `command` (the executable to spawn), defaulting to `"ngrok"`.
  - `spawnAgent(command, port, domain)` spawns the given command. Track `activeCommand` and respawn when it changes (alongside port/domain).
  - The `ENOENT` message names the command that was tried (`ngrok was not found (tried "<command>")…`).
- Main process (`electron/main.cjs`):
  - In `mcp:configure`, read `ngrokUseCustomPath` and `ngrokPath`, and compute the command as `ngrokUseCustomPath && ngrokPath ? ngrokPath : "ngrok"`. Pass `command` to `ensureTunnel`.
- Types (`src/electron.d.ts`): add `ngrokUseCustomPath` and `ngrokPath` to `ConfigureMcpServerInput`.
- App shell (`src/App.tsx`): include `ngrokUseCustomPath` and `ngrokPath` in the `mcp:configure` payload and the effect dependency array.
- Settings dialog (`src/components/settings/SettingsDialog.tsx`):
  - Add a "Use a custom ngrok path" checkbox bound to `mcpServer.ngrokUseCustomPath`, and a conditional "ngrok executable path" text input bound to `mcpServer.ngrokPath`, both routed through `onMcpServerChange`.

## Verification

- Run `npm run build`.
- Run `npm run test` and confirm the updated settings tests pass.
- Run `node --check electron/main.cjs`.
- Run `node --check electron/ngrok-tunnel.cjs`.
- Manual test: with "Use a custom ngrok path" checked and a valid path entered, the tunnel spawns that binary.
- Manual test: with the checkbox unchecked (or checked but the path empty), the tunnel resolves `ngrok` from PATH as before.
- Manual test: a wrong custom path produces a clear "ngrok was not found (tried ...)" error and leaves the local MCP server running.
- Manual test: changing the path while the tunnel is on respawns the agent with the new command.
