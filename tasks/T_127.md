# T_127: Import PDF or image content at the caret

## Status

Done

## Summary

Expand the current AI/Image to Markdown action into an "Import PDF or Images…" workflow. The user
can select one PDF or one or more image files, Nexus captures the current rich-text or source-mode
caret before the picker opens, converts the selected content into ordered Markdown, and inserts the
result at that exact caret after the asynchronous extraction finishes.

## Proposed Behavior

- Replace the single-image picker with a picker that accepts either one PDF or multiple supported
  raster/SVG images while preserving the user's selected file order.
- Reuse the configured vision provider for OCR/layout transcription of standalone images and scanned
  PDF content; preserve the existing clear error when the chosen model has no vision support.
- For text PDFs, extract selectable text locally instead of paying for OCR when possible.
- Extract actual embedded raster images from PDFs locally and emit them as embedded Markdown image
  data URLs near their page's text. The researched `unpdf` package supplies PDF.js-based text and
  image extraction across Node/browser runtimes under a permissive license.
- Insert one combined Markdown result through the existing captured-caret apply path. Keep the
  document unchanged if selection, extraction, or transcription is canceled or fails.
- Show progress for multi-page/multi-file work and enforce explicit page/file/decoded-byte limits so
  a large source cannot silently create an unbounded model request or document.

## Decision

Standalone source images use transcription only: text, tables, structure, and non-text visuals become
Markdown/concise descriptions without embedding the full screenshot or scan. PDFs still preserve
genuinely embedded raster pictures such as photos and figures.

## Implementation Notes

- `electron/documentImport.cjs` validates one-PDF-or-many-images selection, enforces page/file/data
  limits, extracts PDF text and raster images through `unpdf`, renders only image-only pages, and uses
  `@napi-rs/canvas` to encode extracted raw pixels as PNG data URLs.
- `src/lib/documentImport.ts` assembles all ordered sources into one multimodal request and merges
  extracted PDF pictures back into the marked page sections without embedding standalone sources.
- `src/App.tsx` reuses the existing rich/source insertion snapshot and provider handling; the native
  and renderer AI menus now expose "Import PDF or Images…".

## Acceptance Criteria

- One PDF or multiple images can be selected in one import action.
- Imported content lands at the caret that was active before the picker/progress workflow opened in
  rich-text and source modes.
- Text, tables, headings, lists, code, and math are returned as Markdown in source order.
- Embedded PDF pictures are retained as real Markdown images rather than hallucinated links.
- Scanned/image-only content is transcribed through the existing configured vision provider.
- Cancel/failure is non-destructive and reports a concrete reason.
- Automated tests cover picker results, PDF extraction normalization, multi-image request assembly,
  size/count limits, and caret-aware insertion; build, lint, and the full test suite pass.
