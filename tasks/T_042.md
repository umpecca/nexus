# T_042: Keep external reloads clean after programmatic editor updates

**Task Description:** Prevent files reloaded after outside edits from being marked dirty by stale MDXEditor change events.

**Task Status:** Complete

**Task Status Details:** Added a short-lived programmatic Markdown replacement guard in the renderer so stale `onChange` events from the replaced buffer are ignored while the new disk contents become the clean saved baseline.

## Context

After reloading a file that changed outside Nexus, the editor could sometimes appear dirty and prompt Save/Don't Save on close even though the user had accepted the disk version. The reload path updated the saved baseline correctly, but MDXEditor can emit change events during programmatic `setMarkdown()` calls. A late event from the old buffer could overwrite the renderer state after the reload.

## Acceptance Criteria

1. Reloading an externally changed file updates the editor to the disk contents.
2. The reloaded disk contents become the clean saved baseline.
3. Stale `onChange` events from the pre-reload buffer do not mark the document dirty.
4. Real user edits after the reload still mark the document dirty.
5. File/New and File/Open programmatic replacements keep the same stale-event protection.

## Implementation Notes

- Added a `ProgrammaticMarkdownChange` ref that stores the outgoing and incoming Markdown around programmatic document replacement.
- Replaced direct `onChange={setMarkdown}` with a guarded `handleMarkdownChange()`.
- Ignored editor change events equivalent to the outgoing buffer while the programmatic replacement guard is active.
- Kept events equivalent to the incoming buffer aligned with renderer state.
- Cleared the guard automatically after a short timeout or immediately when a real edit arrives.

## Verification

Run `npm run test`, `npm run build`, and Electron CommonJS syntax checks. Manually verify external reload followed by window close does not prompt to save unless the user edits after reload.
