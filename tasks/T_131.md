# T_131: Guide request-body media type selection

## Status

Done

## Verification

## Summary

Replace the free-form request-body media-type field with a guided selector and contextual form help,
so OpenAPI users can choose JSON, URL-encoded forms, multipart uploads, plain text, XML, or a custom
type without memorizing MIME strings.

## Acceptance Criteria

- Request-body media type uses a labelled selector with JSON, URL-encoded form, multipart form/file
  upload, plain text, XML, and Custom choices.
- Existing non-standard media types appear as Custom and retain their exact value until changed.
- Choosing a standard option renames the complete media node, preserving its schema, example,
  extensions, and other unknown fields.
- Custom mode exposes an explicitly labelled media-type input with validation for an empty value.
- Contextual guidance explains when to use each standard choice; multipart guidance includes the
  `string` / `binary` convention for file fields.
- Existing inline fields, component schema selection, required flags, and JSON examples remain
  available without data loss.
- Tests cover media-type option mapping and the custom-value behavior; PRODUCT, ARCHITECTURE,
  CHANGELOG, and this task record are updated.
