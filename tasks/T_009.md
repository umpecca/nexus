# T_009-PowerShell NW.js Runner

**Task Name:** PowerShell NW.js Runner
**Task ID:** T_009
**Task Description:** Add a PowerShell script that runs Nexus as an NW.js application.
**Task Status:** CLOSED
**Task Status Details:** Superseded by T_010. This task originally added an NW.js runner, but the project now uses Electron and `scripts/run-electron.ps1`.

## Problem

The project had an npm script for launching NW.js, but Windows users did not have a direct PowerShell entry point that checked the expected local setup and launched the app consistently.

## Solution

1. Add a `scripts` directory if needed.
2. Add `scripts/run-nw.ps1`.
3. Resolve the repository root relative to the script location.
4. Check for npm, `package.json`, installed dependencies, and the local NW.js launcher.
5. Build by default before launching NW.js.
6. Allow faster launches with `-SkipBuild` and first-run dependency installation with `-InstallIfMissing`.

## Verfication

Superseded by T_010. Run `powershell -NoProfile -ExecutionPolicy Bypass -File scripts/run-electron.ps1 -SkipBuild` to launch an existing Electron build, or omit `-SkipBuild` to build before launch. Run `npm run build` and `npm run test` to verify the project still compiles and tests pass.
