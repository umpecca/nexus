# T_093: Keep PDF export on rich rendered output

## Status

Done

## Summary

Fix the PDF export regression where export could silently fall back to a plain text PDF instead of preserving rendered rich Markdown output.

## Implementation Notes

- Removed the text-first PDF serializer so File/Export as PDF cannot silently write a non-rich fallback PDF.
- Kept PDF export focused on rendered HTML-to-PDF output.
- Changed the final PDF failure mode to report an export error instead of writing a non-rich PDF.
- Superseded the preview-era retry pipeline in T_094 by restoring the direct pre-print-preview export flow.
