# T_090: Scroll active find matches into view

## Status

Done

## Summary

Fix Edit/Find so the active match scrolls into view when a search starts or when the user moves between matches.

## Implementation Notes

- Added a find-panel callback for the active MDXEditor search range.
- Added Nexus-owned range scrolling in the app shell so search navigation uses the currently visible editor scroll container.
- Kept enough top and bottom scroll comfort that the floating find panel does not cover the active match.
- Updated product and architecture notes to require active-match scrolling.
