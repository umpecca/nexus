# T_107: Add Publish as HTML over QuickConnect (HTTP)

## Status

Done

## Summary

Add a second web-publishing target, "Publish as HTML over QuickConnect", alongside the existing SFTP publish. It renders the current document to the same self-contained HTML and sends it as an HTTP POST to a user-configured endpoint with a bearer token and a path header, intended for a simple self-hostable receiving server. The endpoint URL, path, and bearer token are saved per OS profile (the token is persisted by explicit user choice, unlike the never-stored SFTP secrets).

## Implementation Notes

- Settings (`src/lib/settings.ts`):
  - Add a `quickConnect` object to `UserSettings`: `{ url: string; path: string; token: string }`.
  - Add `createDefaultQuickConnect()` (all empty strings) and `sanitizeQuickConnect(value)` (coerce each field to a trimmed string; non-string -> "").
  - Wire `quickConnect` into `createDefaultSettings`, `loadSettings` (sanitized), and the saved object so `saveSettings` round-trips it.
  - The bearer token IS persisted here by explicit user choice; do not treat it like the SFTP secrets.
  - Extend `src/lib/settings.test.ts`: default `quickConnect`, loading a valid stored object, and sanitizing an invalid one. Update the existing full-object `createDefaultSettings`, font-only load, and save assertions to include `quickConnect`.
- Main process (`electron/main.cjs`):
  - Add a `quickconnect:publish` IPC handler. Payload: `{ transport: "quickconnect", currentPath, markdown, options, connection: { url, path, token } }`.
  - Render the page with the existing `renderMarkdownSelfContainedHtml(markdown, currentPath, options)`; do not add a new renderer.
  - Validate non-empty `url` and `path` first; throw a clear error otherwise.
  - POST with the global `fetch`, bounded by `AbortSignal.timeout(30000)`: body = raw HTML, headers `Content-Type: text/html; charset=utf-8`, `Authorization: Bearer <token>`, `X-QuickConnect-Path: <path>`.
  - Treat `response.ok` (2xx) as success. Attempt to read a returned page URL: parse the response body as JSON and use a `url` field, or use the body if it is a plain `http(s)` URL; otherwise return `null`.
  - On a non-2xx response, throw an error that includes the status code and a short snippet of the response body. Catch `AbortError`/timeout and network errors and surface clear messages.
  - Reuse the existing `showPublishErrorForWindow` popup on failure (same pattern as the SFTP handler). Return `{ ok: true, url }` on success and `{ ok: false, error }` on failure.
  - Add a `File / Publish as HTML over QuickConnect…` menu item next to the SFTP publish item, routed via `sendMenuAction("publishQuickConnect")`.
- Preload (`electron/preload.cjs`) and types (`src/electron.d.ts`):
  - Expose `publishQuickConnect(payload)`.
  - Add the QuickConnect payload and result types, and add `"publishQuickConnect"` to the `NexusMenuAction` union.
- Renderer dialog (`src/components/publish/QuickConnectDialog.tsx`):
  - Shadcn-styled dialog with fields for the endpoint URL, path, and bearer token, pre-filled from the saved `quickConnect` settings.
  - A publish action, renderer-side progress while the request is in flight, inline error display, and a success view that shows the returned URL with a copy-to-clipboard control when present.
  - Reuse the `nexus-publish-*` styles. No host/port/username/auth-method or host-key step.
  - Keep the reset-on-open effect keyed on `[open]` only, to avoid the mid-publish form-reset bug fixed in the SFTP dialog.
- App shell (`src/App.tsx`):
  - Handle the `"publishQuickConnect"` menu action to open the QuickConnect dialog, pre-filled from `settings.quickConnect`.
  - On submit, persist `url`, `path`, and `token` back into `settings.quickConnect`, and send markdown + current file path + export options + connection to `window.nexus.publishQuickConnect`.
