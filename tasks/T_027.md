# T_027: Add profile-scoped editor font settings

**Task Description:** Add a shadcn-styled settings dialog that lets the user choose the editor font and persists that preference locally per OS profile name.

**Task Status:** Completed

**Task Status Details:** Added a Settings/Preferences menu item, a Radix-backed shadcn-style dialog, profile-name lookup through Electron IPC, localStorage settings helpers, and editor font application for rich-text and source editor surfaces.

## Product Context

Nexus should feel like a desktop application, including a native settings entry point for personal editor preferences. Font choice is a user preference rather than document content, so it should persist locally for the current OS profile.

## Acceptance Criteria

1. The app includes a Settings/Preferences menu item.
2. Activating Settings/Preferences opens a shadcn-styled modal dialog.
3. The dialog allows the user to choose the editor font.
4. The chosen font applies to the editor content area.
5. The chosen font persists in localStorage under a key scoped to the current OS profile name.
6. The app continues to build and tests continue to pass.

## Implementation Notes

- Added `@radix-ui/react-dialog` and local shadcn-style dialog primitives.
- Added `SettingsDialog` with editor font selection and preview.
- Added `src/lib/settings.ts` for default settings, profile-scoped localStorage keys, and safe load/save behavior.
- Added Electron IPC for retrieving `os.userInfo().username` from the main process.
- Applied the selected font through an editor-surface CSS variable.

## Verification

Run `npm run build` and `npm run test`. Launch the app, open Settings/Preferences, choose a font, close and reopen the app, and confirm the selected font remains active for the same OS profile.
