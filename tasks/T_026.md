# T_026: Remove duplicate undo and redo toolbar controls

**Task Description:** Remove undo and redo from the editor toolbar because those commands are already available in the native Electron Edit menu.

**Task Status:** Completed

**Task Status Details:** Removed MDXEditor's `UndoRedo` toolbar control from the shadcn-styled toolbar and updated product, architecture, and changelog documentation to treat undo/redo as native menu actions.

## Product Context

Nexus should feel like a desktop application with common editing commands in the native menu. Duplicating undo and redo in the editor toolbar creates visual clutter and repeats controls already provided by the menubar.

## Acceptance Criteria

1. The editor toolbar no longer displays undo or redo buttons.
2. Edit/Undo and Edit/Redo remain available through the native Electron Edit menu.
3. The toolbar still exposes the remaining MDXEditor formatting, insert, block, and source controls.
4. The app build and test suite continue to pass.

## Implementation Notes

- Removed the `UndoRedo` import and toolbar group from `src/components/editor/ShadcnMdxToolbar.tsx`.
- Kept native menu undo/redo behavior in Electron unchanged.
- Updated AI-DLC documentation to describe undo/redo as menu-only actions.

## Verification

Run `npm run build` and `npm run test`. Launch the app and confirm the toolbar starts with text formatting controls while Undo and Redo remain in the Edit menu.
