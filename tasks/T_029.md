# T_029: Keep rich/source editor scroll positions in sync

**Task Description:** Preserve the user's current document position when switching between MDXEditor's rich text and source views.

**Task Status:** Completed

**Task Status Details:** Added renderer-side scroll synchronization that tracks the active editor scroll container and reapplies the current normalized scroll position when MDXEditor swaps between rich text and source view containers.

## Product Context

Switching between rich text and source mode should feel like changing the representation of the same document, not jumping to another location. A user halfway down a document in rich text should land halfway down the source view, and the same should happen when switching back.

## Acceptance Criteria

1. When the user scrolls in rich text mode and switches to source mode, the source editor opens at the corresponding document position.
2. When the user scrolls in source mode and switches to rich text mode, the rich text editor opens at the corresponding document position.
3. The synchronization does not require a separate visual/source tab bar.
4. Existing save, settings, and menu workflows continue to work.
5. The app build and test suite continue to pass.

## Implementation Notes

- Added editor surface tracking in `src/App.tsx`.
- Captured scroll position as both absolute `scrollTop` and normalized ratio.
- Observed MDXEditor's active rich/source scroll containers with `MutationObserver`.
- Applied the stored scroll ratio to the newly active container after view changes.

## Verification

Run `npm run build` and `npm run test`. Launch the app, scroll midway through a long document, switch to source mode, and confirm the source editor appears at the corresponding position. Repeat in reverse from source mode back to rich text.
