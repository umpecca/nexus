# T_060: Align visual paper margins with first block spacing

**Task Status:** Completed

**Task Description:** Fix the visual editor so configured page margins measure to visible content instead of being inflated by the first or last Markdown block's default margins.

## Context

When all paper margins are set to 0.25 inches, the left padding is correct but the top can look larger because headings and paragraphs bring their own default top margin inside the paper padding.

## Acceptance Criteria

1. The first visible rich-text block starts at the configured top page margin.
2. The last visible rich-text block does not add extra apparent bottom page margin.
3. Internal document spacing remains normal.
4. Paper and plain rich-text views continue to work.
5. Source mode, diff mode, HTML export, and PDF export behavior are unchanged.

## Implementation Notes

- Removed top margin from the first direct child of `.markdown-body`.
- Removed bottom margin from the last direct child of `.markdown-body`.
- Kept the correction scoped to rendered Markdown content.

## Verification

Run `npm run test`, `npm run build`, and `git diff --check`. Manually set all margins to 0.25 and confirm the first text aligns visually with the configured top and left margins.
