# T_063: Add a plain-view responsive content wrapping toggle

**Task Status:** Completed

**Task Description:** Add a saved toolbar control that lets users turn responsive content wrapping on or off when the rich-text editor is in plain view mode.

## Context

Plain view removes the paper sheet and can either use a readable centered column or wrap content to the full application width. The wrapping toggle should never create a page-level horizontal scrollbar.

## Acceptance Criteria

1. A toolbar control toggles responsive content wrapping for plain view.
2. The wrapping control is not active while paper view is enabled.
3. The wrapping preference is stored in profile-scoped settings.
4. Existing settings load safely, defaulting responsive content wrapping to enabled.
5. Responsive content wrapping in plain view wraps to the width of the application without creating a page-level horizontal scrollbar.

## Implementation Notes

- Added `responsiveContentWrappingEnabled` to the persisted settings model.
- Added an icon-only responsive wrapping toolbar toggle in the view controls group.
- Added a plain-mode responsive wrapping CSS branch that expands the editable surface to application width while preserving soft wrapping.
- Added settings tests for default, invalid, saved, and serialized wrapping preferences.
