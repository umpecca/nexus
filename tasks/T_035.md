# T_035: Replace Sandpack with a local JavaScript runner

**Task Status:** Completed

**Task Description:** Remove Nexus's Sandpack/CodeSandbox editor functionality and replace it with local JavaScript runner blocks that execute browser JavaScript inside the app without using a third-party runtime service.

## Context

Sandpack blocks depend on CodeSandbox-hosted runtime infrastructure. Nexus should keep runnable examples local-first, so this task removes the app's Sandpack plugin and toolbar actions while keeping MDXEditor as the underlying editor.

## Acceptance Criteria

1. Nexus no longer registers `sandpackPlugin`.
2. The toolbar no longer uses `InsertSandpack` or `ShowSandpackInfo`.
3. The toolbar includes an Insert JS runner action.
4. Insert JS runner creates a fenced JavaScript code block with `nexus-run` metadata.
5. `js nexus-run` and `javascript nexus-run` code blocks render with CodeMirror editing plus a Run panel.
6. Running a block executes locally in a temporary Web Worker and displays console output or errors.
7. Runner execution blocks common network and nested worker APIs and times out long-running snippets.
8. Plain JavaScript code blocks remain normal non-runnable code blocks.

## Implementation Notes

- Added a local JavaScript runner utility with worker-source generation, console capture, timeout handling, and runnable block matching.
- Added a custom MDXEditor code block descriptor for JavaScript blocks whose meta contains `nexus-run`.
- Added project-owned toolbar insertion for local runner blocks.
- Removed Nexus's Sandpack plugin and toolbar components while leaving MDXEditor's transitive package dependencies untouched.

## Verification

Run `npm run test` and `npm run build`. Launch the app, insert a JS runner block, run it, and confirm local console output appears without opening a CodeSandbox/Sandpack runtime.
