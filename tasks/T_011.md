# T_011-Electron File Menu Actions

**Task Name:** Electron File Menu Actions
**Task ID:** T_011
**Task Description:** Add File/New, File/Open, File/Save, and File/Save As menu items to the Electron app menu and implement their functionality.
**Task Status:** CLOSED
**Task Status Details:** Added a native Electron File menu, forwarded menu actions to the renderer, exposed safe preload APIs for Markdown file open/save operations, and updated renderer document state to track the current file path.

## Problem

The Electron app had no native document menu, so users could not create, open, or save Markdown documents through standard desktop menu actions.

## Solution

1. Build an Electron application menu with File/New, File/Open, File/Save, and File/Save As.
2. Send menu actions from the main process to the focused renderer window.
3. Add preload APIs for opening a Markdown file, saving to the current path, and saving with a dialog.
4. Implement main-process file dialogs and disk I/O with Electron IPC handlers.
5. Track the current file path in renderer state and local draft persistence.

## Verfication

Run `npm run build` and `npm run test`. Launch the Electron app and manually verify File/New resets the draft, File/Open loads a Markdown file, File/Save writes to the current file path or prompts when there is none, and File/Save As prompts for a destination.
