# T_138: Add toggleable WYSIWYG page boundaries

## Status

Done

## Summary

Extend the existing rich-text Paper View with an optional, persisted page-boundary overlay. The
guides use the configured paper size, orientation, margins, and editor zoom to show where pages end
without inserting page-break markup or otherwise changing the document Markdown.

## Acceptance Criteria

1. The View menus expose a **Show Page Boundaries** toggle while Paper View is enabled.
2. The preference is stored per OS profile and safely defaults for older settings.
3. Rich-text Paper View shows repeated page-ending guides at the configured page height.
4. Plain, source, and diff modes remain unchanged, and the guides do not affect saved Markdown or
   document exports.
5. Tests, lint, build, and Electron main-process syntax checks pass.

## Implementation Notes

- Added the `pageBoundariesEnabled` per-profile setting with a safe `false` migration default.
- Added **Show Page Boundaries** to both rendered and native View menus. It is available only when
  Paper View and rich-text mode are active.
- The rich-text content surface uses a repeating background rule based on
  `--editor-page-height`, which already reflects paper size, orientation, and editor zoom. The guide
  never creates editor nodes or changes Markdown/export output.
- Updated product, architecture, and changelog documentation.
- Completed `npm run test` (564 tests), `npm run lint`, `npm run build`, and
  `node --check electron/main.cjs`.
