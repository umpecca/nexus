# T_105: Add an outline / table-of-contents sidebar

## Status

Done

## Summary

Add a collapsible outline sidebar beside the rich-text editor that lists the current document's headings (H1–H6) as a depth-indented, clickable tree. Selecting an entry scrolls the rich-text editor to that heading. The sidebar updates as headings change, shows an empty-state message when the document has no headings, is shown only in rich-text mode (hidden in source and diff modes), can be toggled from the editor toolbar, and stores its open/closed state per OS profile. Default is hidden.

## Implementation Notes

- Add `src/lib/outline.ts` that extracts an ordered list of headings from the current Markdown buffer.
  - Return an array of `{ level: 1..6, text: string, index: number }`, where `index` is the heading's zero-based ordinal position across all headings in document order.
  - Support ATX headings (`#`..`######`). Trim the heading text and strip trailing `#` markers.
  - Ignore headings inside fenced code blocks (``` and ~~~ fences), matching the export renderer's fence handling, so code comments are not treated as headings.
  - Keep the helper pure and dependency-free so it is unit testable.
- Add `src/lib/outline.test.ts` covering: nested levels, no-heading documents, headings ignored inside fenced code blocks, duplicate heading text producing distinct ordinal indexes, and trimming.
- Add an `outlineVisible` boolean to the settings model in `src/lib/settings.ts`.
  - Default to `false` (hidden).
  - Extend the settings sanitizer/loader to coerce non-boolean stored values back to the default, consistent with the existing per-field sanitizers.
  - Store it inside the existing `nexus:settings:v1:{profileName}` object; do not add a new storage key.
- Extend `src/lib/settings.test.ts` with cases for the new field's default and sanitization.
- Add `src/components/editor/OutlineSidebar.tsx`:
  - A shadcn-styled panel that renders the headings from `src/lib/outline.ts` as a clickable list indented by `level`.
  - Accept the heading list and an `onSelect(index)` callback as props.
  - Render a quiet empty-state message (for example, "No headings yet") when the list is empty.
  - Keep it presentational; heading extraction and scrolling live in the app shell.
- Add a toolbar toggle in `src/components/editor/ShadcnMdxToolbar.tsx`:
  - Place an outline visibility toggle button in the view controls group next to the existing paper/plain, orientation, and responsive wrapping toggles, using a Lucide icon (for example, `ListTree` or `PanelLeft`) and a tooltip.
  - Surface it through the existing toolbar prop pattern (`outlineVisible` + `onOutlineVisibleChange`).
- Wire the feature in `src/App.tsx`:
  - Recompute the heading list from the current Markdown buffer when it changes.
  - Render `OutlineSidebar` to the left of the editor surface only when `settings.outlineVisible` is true and the current view mode is rich-text; do not render it in source or diff modes.
  - On entry selection, scroll the matching heading into view by locating the heading element at the selected ordinal position inside the active rich-text editor scroll container, reusing the existing editor scroll helpers used by find navigation (`getEditorScrollElements` / scroll-into-view logic).
  - Persist `outlineVisible` through the existing settings save flow.
- Add outline sidebar styles in `src/styles.css`:
  - A fixed-width, independently scrolling column that follows the existing light/dark theme tokens.
  - Indentation by heading level, hover/active affordances consistent with toolbar controls, and an empty-state style.
  - Ensure the editor surface keeps the remaining width and that the sidebar does not introduce page-level horizontal scrolling.
