# T_120: Add an MCP verification endpoint and a "Test setup" button

## Status

Done

## Summary

Give the user a one-click way to confirm the embedded MCP server (and the optional ngrok tunnel) is
actually reachable and that the bearer token is accepted, instead of wiring up an external client to
find out. Add a lightweight authenticated `GET /health` endpoint to the server, and a "Test setup"
button in the MCP section of Preferences that probes the loopback endpoint and — when the tunnel is
connected — the public ngrok URL too, reporting clear pass/fail for each. The endpoint enforces the
bearer token via the Authorization header exactly like `/mcp` (no query-param token), so a passing
local probe proves auth works and a passing ngrok probe proves the tunnel forwards correctly.

## Implementation Notes

- `electron/mcp-server.cjs`:
  - In `handleRequest`, after OPTIONS and before the `/mcp` path check, parse the URL and handle
    `GET /health`: reuse the same loopback bind and bearer check (`extractBearerToken` +
    `compareTokens`) as `/mcp`, returning `{ ok, name, version, protocolVersion, authMode, time }` on
    success, `401` on a bad/missing token, and `405` for non-GET. Added `GET` to the OPTIONS
    allow-methods.
  - Add `testConnection({ ngrokUrl })` (exported) plus an `httpGetJson(url, token, timeoutMs)` helper
    (`node:http`/`node:https`, Authorization header, timeout → `{ ok, status?, error? }`).
    `testConnection` reads the active token from `currentConfig` (so it never leaves the module),
    probes `http://127.0.0.1:{listeningPort}/health`, and probes `{ngrokUrl}/health` when a URL is
    given, returning `{ local, ngrok }`.
- `electron/main.cjs`: `ipcMain.handle("mcp:test-connection", ...)` calls `mcpServer.testConnection`
  with the public URL from `ngrokTunnel.getTunnelState()` when the tunnel is connected.
- `electron/preload.cjs`: expose `testMcpConnection()`.
- `src/electron.d.ts`: add `McpConnectionProbe`, `McpConnectionTestResult`, and `testMcpConnection()`.
- `src/components/settings/SettingsDialog.tsx`: a new `onTestMcpConnection` prop; `mcpTesting` /
  `mcpTestResult` state with a `useEffect` that clears the result when the dialog closes or the MCP
  reachability settings change; a "Test setup" button (shown when the server is enabled) and a
  pass/fail readout (local always, ngrok when present) via a `describeProbe` helper.
- `src/App.tsx`: pass `onTestMcpConnection={() => window.nexus?.testMcpConnection() ?? Promise.resolve(undefined)}`.
- `src/styles.css`: add `.nexus-settings-success` (green counterpart to `.nexus-settings-warning`) and
  small layout classes for the test block.

## Verification

- `npm run build` (`tsc --noEmit` + vite) and `npm run test` pass (270 tests).
- `electron/mcpServer.health.test.ts` (new): boots the real server and asserts `GET /health` →
  `200` + identity body with a valid token, `401` with a wrong/missing token, `405` for POST, and
  `200` without a token when `authMode` is `none`; and that `testConnection` reports the local probe
  `ok` and exercises the ngrok branch (pointing the "ngrok" URL back at the loopback server).
- `node --check` on the edited `.cjs` files.
- Manual (desktop): enable the MCP server → Test setup shows local ✓; enable the ngrok tunnel →
  Test setup shows local ✓ and ngrok ✓ with the public URL; stop the server or use a stale token →
  the probe shows ✗ with the HTTP status or error.
