# T_031: Prompt after selecting a file to open

**Task Description:** Change File/Open so the dirty-buffer confirmation appears only after the user selects a file in the native open dialog.

**Task Status:** Completed

**Task Status Details:** Reordered the File/Open renderer flow to show the native file picker first, return immediately if the picker is canceled, and then ask whether to save, discard, or cancel only before replacing the current dirty buffer with the selected file.

## Product Context

The native open-file dialog already has its own Cancel button. Nexus should not ask the user what to do with unsaved editor changes when the user cancels that dialog, because no replacement document has been selected.

## Acceptance Criteria

1. If the user chooses File/Open and cancels the native file picker, no dirty-buffer confirmation appears.
2. If the user chooses a file while the current buffer is clean, Nexus opens the selected file.
3. If the user chooses a file while the current buffer is dirty, Nexus prompts Save, Don't Save, or Cancel before replacing the current buffer.
4. Choosing Save saves the current buffer and then opens the selected file.
5. Choosing Don't Save opens the selected file without saving the current buffer.
6. Choosing Cancel keeps the current buffer and does not open the selected file.
7. The app build and test suite continue to pass.

## Implementation Notes

- Moved `openMarkdownFile()` before `confirmDirtyBufferAction()` in `src/App.tsx`.
- Kept the replacement logic behind the existing dirty-buffer confirmation result.
- Updated product and architecture documentation to describe the picker-first File/Open flow.

## Verification

Run `npm run build` and `npm run test`. Launch the app, make the buffer dirty, choose File/Open, cancel the native picker, and confirm no save prompt appears. Repeat by selecting a file and confirm the save/discard/cancel prompt appears before replacement.
