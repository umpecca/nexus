# T_033: Open OS-handed files from Finder or Explorer

**Task Description:** Allow Nexus to open Markdown/text files handed to the app by macOS Finder or Windows Explorer.

**Task Status:** Completed

**Task Status Details:** Added operating-system file handoff handling in Electron, including macOS `open-file`, initial process arguments, and Windows second-instance arguments. Accepted files open in their own editor windows and load through a preload initial-file API.

## Product Context

Users expect a desktop editor to accept files launched from the operating system. On macOS this commonly arrives through Finder's `open-file` event. On Windows this commonly arrives through Explorer/Open With as process arguments, including while another Nexus instance is already running.

## Acceptance Criteria

1. A supported file path passed at app startup opens in a Nexus editor window.
2. macOS Finder `open-file` events are accepted before or after the app is ready.
3. Windows Explorer/Open With file paths passed to a second Nexus process are handed to the running app.
4. Each OS-opened file opens in its own editor window.
5. Unsupported or non-file arguments are ignored.
6. Existing File/Open behavior remains focused-window scoped.
7. The app build and test suite continue to pass.

## Implementation Notes

- Added a single-instance lock and `second-instance` handler for process-argument file handoff.
- Added an `open-file` handler for macOS Finder integration.
- Added Markdown/text file path filtering for `.md`, `.markdown`, `.mdx`, and `.txt`.
- Added pending initial-file payload storage keyed by webContents id.
- Added `getInitialOpenFile()` to preload and renderer typings.
- Updated the renderer to load an initial OS-opened file into the window document state.

## Verification

Run `npm run build`, `npm run test`, and `node --check .\electron\main.cjs`. Launch Electron with a Markdown file path argument and confirm the file opens in its own window. On a packaged app, use Finder or Explorer/Open With and confirm the handed-off file opens in Nexus.
