# T_106: Add Publish as Web over SFTP

## Status

Done

## Status Details

Implemented the settings model + sanitizer/tests, the main-process `sftp:publish` flow with host-key fingerprint confirmation (mirroring the MCP confirm round trip), the preload/types bridge, the `PublishWebDialog`, App wiring, and styles. Deviation from the original notes: progress and errors are shown inside the Publish as Web dialog (renderer-side) rather than through the modal `withExportProgressWindow`, because the host-key fingerprint prompt is itself a renderer dialog and a modal main-process progress window would block the user from accepting it. Build, tests (48 passing), and `node --check` on both electron files pass.

## Summary

Add a File / Publish as Web action that renders the current document to the same self-contained HTML produced by HTML export (inline base64 images, inline bundled fonts, base64 SVG Mermaid) and uploads that single file to a user-specified SFTP server. Credentials are entered on every publish and never stored. Only non-secret connection fields persist per OS profile to pre-fill the dialog. Before any document data is sent, the user must accept the server's host-key fingerprint. SFTP is the only transport in this version.

## Implementation Notes

- Settings (`src/lib/settings.ts`):
  - Add a non-secret `publishTarget` object to `UserSettings`: `host` (string), `port` (number, default `22`), `username` (string), `remoteDirectory` (string), `publicBaseUrl` (string). Defaults empty strings except `port: 22`.
  - Add `createDefaultPublishTarget()` and `sanitizePublishTarget(value)`: coerce each field to a string, default `port` to 22 and clamp it to an integer in `1..65535`, and never read or write any secret field.
  - Wire `publishTarget` into `createDefaultSettings`, `loadSettings` (sanitized), and the saved object so `saveSettings` round-trips it.
  - Extend `src/lib/settings.test.ts`: default `publishTarget`, loading a valid stored target, and sanitizing an invalid one (bad port, non-string fields). Update the existing full-object `createDefaultSettings`, font-only load, and save assertions to include `publishTarget`.
- Main process (`electron/main.cjs`):
  - Add an `sftp:publish` IPC handler. Payload: `{ transport: "sftp", currentPath, markdown, options, connection: { host, port, username, remoteDirectory, remoteFilename, publicBaseUrl }, auth: { kind: "password", password } | { kind: "key", privateKeyPath, passphrase? } }`. The `transport` discriminator keeps room for a future target without reworking the flow.
  - Render the page with the existing `renderMarkdownSelfContainedHtml(markdown, currentPath, options)`; do not add a new renderer.
  - Connect with `ssh2-sftp-client`. Pass an `ssh2` `hostVerifier` that computes the host-key fingerprint as OpenSSH-style `SHA256:` + base64 of the key's SHA-256 digest (Node `crypto`), sends it to the requesting renderer over `sftp:confirm-host-key` with a generated request id, and awaits an accept/reject reply over `sftp:host-key-decision`. Mirror the existing `mcp:confirm-write` / `mcp:write-decision` pending-map pattern (a `Map` of pending resolvers keyed by request id, cleaned up on window close). Only continue the connection on accept; on reject, abort before any upload.
  - When `auth.kind === "key"`, read the private key from `privateKeyPath` in the main process at connect time; never send key contents back to the renderer or persist them.
  - Upload the rendered HTML to `remoteDirectory` + `remoteFilename` (create/ensure the directory path as needed), then end the connection in a `finally`.
  - Reuse `withExportProgressWindow` for progress and the existing export error dialog (`showExportErrorForWindow`) for failures. Return `{ canceled }` on host-key rejection and `{ published: true, url }` on success, composing `url` from `publicBaseUrl` + `remoteFilename` when `publicBaseUrl` is set.
  - Do not log secrets. Keep the document file path, saved baseline, and dirty state unchanged.
  - Add a `File / Publish as Web…` menu item near the export items that calls `sendMenuAction("publishWeb")`.
  - Add a key-file picker IPC handler (`dialog:select-private-key`) used by the renderer.
- Preload (`electron/preload.cjs`) and types (`src/electron.d.ts`):
  - Expose `publishWeb(payload)`, `onConfirmHostKey(cb)` (returns an unsubscribe), `resolveHostKey(requestId, decision)`, and `selectPrivateKeyFile()`.
  - Add matching method signatures and payload/result types to the `window.nexus` declaration.
- Renderer dialog (`src/components/publish/PublishWebDialog.tsx`):
  - Shadcn-styled dialog with connection fields (host, port, username, remote directory, remote filename, optional public base URL), an auth-method toggle (Password | Private key file), password or key path + passphrase inputs, and a key-file picker button.
  - Default `remoteFilename` to a slug derived from the document title/frontmatter (small local helper producing `my-doc.html`).
  - Show a host-key confirmation step that displays the fingerprint with Accept and Reject actions; show progress while connecting/uploading; show errors inline; show a success view with the composed URL and a copy-to-clipboard control when available.
  - Keep all secret values in component state only; never write them to settings.
- App shell (`src/App.tsx`):
  - Handle the `"publishWeb"` menu action to open the dialog, pre-filled from `settings.publishTarget`.
  - On submit, persist only the non-secret connection fields back into `settings.publishTarget`, and send markdown + current file path + export options + connection + auth to `window.nexus.publishWeb`.
  - Wire the host-key confirmation round trip (`onConfirmHostKey` -> dialog -> `resolveHostKey`).
- Styles (`src/styles.css`): add Publish as Web dialog styles for light and dark themes.
