# T_010-Electron Runtime Migration

**Task Name:** Electron Runtime Migration
**Task ID:** T_010
**Task Description:** Convert Nexus from an NW.js desktop runtime to an Electron desktop runtime.
**Task Status:** CLOSED
**Task Status Details:** Added Electron main and preload files, changed package metadata and scripts to launch Electron, replaced the NW.js runner with an Electron runner, and updated documentation to describe Electron as the desktop shell.

## Problem

Nexus was scaffolded as an NW.js application, but Electron is a better fit for the project because its desktop runtime, security model, packaging ecosystem, and native integration patterns are more widely used and better documented.

## Solution

1. Add an Electron main process file that creates the desktop window.
2. Add a minimal preload file for future safe renderer-to-main APIs.
3. Change `package.json` so Electron owns the app entry point.
4. Replace NW.js launch scripts with Electron launch scripts.
5. Replace the PowerShell NW.js runner with an Electron runner.
6. Update README, product, architecture, and changelog references from NW.js to Electron.

## Verfication

Run `npm install` to update runtime dependencies. Run `npm run build` and `npm run test`. Run `powershell -ExecutionPolicy Bypass -File scripts/run-electron.ps1 -SkipBuild` to launch the built Electron app.
