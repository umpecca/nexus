# T_007-Remove Top Document Actions

**Task Name:** Remove Top Document Actions
**Task ID:** T_007
**Task Description:** Remove the top section containing the New, Import, and Export buttons from the current app shell.
**Task Status:** CLOSED
**Task Status Details:** Removed the topbar, file name state, document action buttons, hidden file input, import/export handlers, Markdown download helper, and related styles. Updated product and architecture docs to defer file workflows.

## Problem

The top document action section added file-management controls before the current iteration needed them, making the app feel less focused on the editor workspace.

## Solution

1. Remove the header section containing the file name and New, Import, and Export buttons.
2. Remove unused file name, file input, import, export, and new-document code paths.
3. Simplify draft persistence to store only Markdown content.
4. Remove styles that only supported the top action section.
5. Update AI-DLC docs to describe file actions as deferred future work.

## Verfication

Run `npm run build` and `npm run test`. Open the app and confirm the first visible surface is the MDXEditor workspace with no top New, Import, or Export section.
