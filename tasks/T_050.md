# T_050: Replace the editor toolbar with an Office-inspired grouped toolbar

**Task Name:** Replace the editor toolbar with an Office-inspired grouped toolbar

**Task Description:** Replace the tabbed ribbon toolbar with a Nexus-owned, Microsoft Office-inspired single-band toolbar that shows editor command groups at once while preserving MDXEditor command wiring and using a light gray Office-style palette.

**Task Status Details:** Removed the Home/Insert/View tab row, removed decorative Nexus badge chrome, removed the toolbar refresh command, laid out formatting, paragraph, insert, and view mode controls together in labeled groups, restored the lighter gray ribbon chrome, floated the mode controls to the right, centered the Links & Media controls, added white bordered paragraph dropdown controls, added consistent tooltip wrappers to Nexus-owned toolbar buttons, raised and transform-offset dropdown/tooltip layers so they clear the sticky toolbar, and kept the outer toolbar borders thin and flush with the app edge.

## Problem Statement

The functional tabbed ribbon matched the Office reference but hid commands behind tab switching. Nexus needs the same compact desktop style while keeping the toolbar direct: editor commands should be visible together, grouped by section label, and allowed to overflow horizontally at narrow widths instead of wrapping into misaligned rows. Document actions such as refresh should remain in native menus.

## Acceptance Criteria

1. The editor toolbar renders as a single Office-inspired toolbar band without Home, Insert, or View tabs.
2. Formatting, paragraph, insert, and mode controls are visible together in labeled groups in rich text mode.
3. Source and diff modes keep the toolbar visible with the view mode controls available.
4. Undo, redo, and document file actions remain in the native Electron menus.
5. Refresh remains available through the native Edit menu, not the toolbar.
6. The toolbar uses a light gray Office-style color scheme with yellow hover and active states.
7. Narrow widths use horizontal command overflow instead of wrapped misaligned toolbar rows.
8. The toolbar remains sticky and does not overlap editable document content.
9. Toolbar buttons expose consistent tooltips, including Nexus-owned insert and view mode controls.
10. Toolbar dropdowns, including the paragraph/block type dropdown, render above the sticky toolbar.
11. The command band has subtle horizontal gray borders for clearer separation without left/right edge borders.
12. The mode group is visually anchored to the right side of the toolbar when horizontal space is available.
13. The paragraph block type dropdown uses a white background with a visible gray border.
14. The Links & Media group centers its compact controls within the group.
15. Dropdowns, popovers, and tooltips opened from toolbar controls are not visually covered by the sticky toolbar.

## Implementation Notes

- Implement the grouped toolbar directly in `ShadcnMdxToolbar.tsx` with no local tab state and no new package dependency.
- Keep MDXEditor command components as the interactive controls inside labeled groups to preserve editor command behavior.
- Scope styling under `.nexus-office-ribbon` so the light Office-inspired palette and bevels do not leak into unrelated UI.
- Use MDXEditor's `TooltipWrap` for Nexus-owned toolbar buttons so custom controls match built-in toolbar tooltip behavior.
- Use small toolbar group modifier classes for right-aligned modes and centered media controls rather than changing MDXEditor command components.
- Keep MDXEditor Radix popper wrappers above the sticky toolbar and use transform-based bottom-side clearance so toolbar dropdowns and tooltips open below the ribbon body.
