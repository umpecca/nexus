# T_104: Reimplement Export as HTML as a self-contained document

## Status

Done

## Summary

Rebuild File / Export as HTML so the saved `.html` file is portable when moved away from the original Markdown folder or the Nexus installation. The export should remain a static rendered Markdown document, but supported local images, Mermaid diagrams, and bundled web font files should be embedded directly with base64 `data:` URLs.

## Implementation Notes

- Add a self-contained HTML export path in `electron/main.cjs` separate from PDF's existing print-oriented flow.
- Resolve relative and absolute local Markdown image sources, including `file://` image URLs, and rewrite supported image tokens to `data:<mime>;base64,...` before Marked renders the document.
- Keep remote image URLs and unsupported image extensions unchanged rather than attempting network-dependent export work.
- Render fenced `mermaid` blocks once in a hidden BrowserWindow and replace successful diagrams with `<img src="data:image/svg+xml;base64,...">` elements.
- Preserve inline Mermaid error blocks for invalid diagram syntax.
- Inline selected bundled Fontsource CSS and referenced font files as base64 font data URLs for HTML export, while leaving PDF's existing font-loading behavior unchanged.
- Load the self-contained HTML into the hidden Mermaid render window from a temporary file so large base64 image/font payloads are not routed through a navigation data URL.
- Keep File / Export as HTML from changing the active document path, saved baseline, or dirty state.
