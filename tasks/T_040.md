# T_040: Preserve pre-reload versions for diff review

**Task Description:** Preserve the current editor contents before reloading an externally changed file so MDXEditor diff mode can compare the previous in-memory version with the newly loaded disk version.

**Task Status:** Complete

**Task Status Details:** Updated reload handling to capture the active editor Markdown before disk reload, store it as the previous-version baseline, and point the diff baseline at that preserved content.

## Context

Nexus can already review dirty external conflicts before a reload, but after choosing Reload the replaced editor contents were no longer available as a comparison baseline. A desktop-style workflow should allow the user to reload the new file and still compare it against the version that was in the editor immediately before the reload.

## Acceptance Criteria

1. When a watched file change prompt is reloaded, Nexus captures the current editor Markdown before replacing it.
2. The file reload still updates the editor to the latest disk contents and clears the dirty marker.
3. The captured pre-reload Markdown becomes the previous-version diff baseline.
4. Edit/Compare with Previous Version compares the reloaded disk contents against the preserved pre-reload version.
5. Successful saves continue to preserve the pre-save version as the previous-version baseline.
6. New/opened documents without a previous baseline still no-op for previous-version comparison.

## Implementation Notes

- Renamed the renderer baseline state from previous saved content to previous version content.
- Extended `loadDocument()` with an optional previous-version baseline used by external-change reloads.
- Set `diffMarkdown` to the preserved baseline so MDXEditor's diff mode compares previous contents against the newly loaded document.
- Renamed the native Edit menu item and renderer action from previous save to previous version.
