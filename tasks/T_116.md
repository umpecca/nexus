# T_116: Add a source-mode "Clean up formatting" command

## Status

Done

## Summary

Documents in Nexus are increasingly co-edited by humans and AI agents, which leaves the raw Markdown
inconsistently formatted: mixed `-`/`*`/`+` list markers, headings jammed against surrounding text,
ragged table pipes, stray blank-line runs, and trailing whitespace. This adds a "Clean up formatting"
command — a brush button that appears only while editing in source mode — that normalizes the raw
Markdown in place: unordered markers and marker spacing, one blank line above and below ATX headings,
re-padded/aligned GFM tables, `---` thematic breaks, collapsed blank-line runs, and trimmed trailing
whitespace (two-space hard breaks preserved). Fenced code blocks and a leading YAML frontmatter block
are passed through verbatim, and the pass is idempotent.

The command is deliberately source-mode-only (per the request): it operates on the raw Markdown text,
so it is offered alongside the floating view controls that source mode already shows, and is hidden in
rich-text and diff modes.

## Implementation Notes

- New `src/lib/format.ts`: pure `cleanupMarkdownFormatting(markdown)`. Tokenizes the body (fence- and
  table-aware), applying per-line transforms — `normalizeListMarker` (unordered → `- `, collapse
  marker spacing, ordered numbers preserved), ATX heading normalization (one space after the hashes,
  trailing `###` closer stripped, leading indent dropped), `isThematicBreak` → `---`, and
  `trimTrailing` (keeps a normalized two-space hard break) — then a structural `normalizeTokens` pass
  that collapses blank runs, guarantees a single blank line around headings, and trims edge blanks.
  GFM tables are parsed (`splitRowCells` respects `\|`, `parseDelimiterRow` reads alignment) and
  re-rendered with per-column widths via `formatTableBlock`/`padCell`/`delimiterCell`. Fenced code
  uses a char+length open/close matcher so a longer outer fence is not closed by a shorter inner one.
  A non-heading hash run with no following space (`##Section`) is left as a paragraph, matching
  CommonMark, so the cleanup never changes a document's meaning.
- New `src/lib/format.test.ts`: 27 vitest cases across list markers, headings, tables (alignment,
  ragged/pipeless rows, escaped pipes, setext-underline safety), thematic breaks, blank-line and
  trailing-whitespace handling, CRLF normalization, fenced-code/frontmatter pass-through, and a
  kitchen-sink idempotency check.
- New `src/components/editor/CleanUpFormatting.tsx`: a `TooltipWrap` + ghost `Button` (lucide `Brush`),
  mirroring `InsertTableOfContents.tsx`.
- `src/components/editor/ShadcnMdxToolbar.tsx`: thread an `onCleanUpFormatting` prop through; pass the
  current `viewMode` to `ViewRibbonCommands`, which renders `CleanUpFormatting` (plus a separator)
  inside the floating mode-controls cluster only when `currentMode === "source"`.
- `src/App.tsx`:
  - Extend the `SourceEditorView` type and `getSourceEditorView` guard (which already reaches
    MDXEditor's CodeMirror view via the content node's `cmTile.view` for the outline) to also expose
    `dispatch` and `doc.length`/`doc.toString()`.
  - Add `cleanUpFormatting`: compute `cleanupMarkdownFormatting(getCurrentMarkdown())`; if it differs,
    replace the CodeMirror buffer in place with a single `dispatch` change and mirror the result into
    React state. This bypasses `setMarkdown`, which would re-run MDXEditor's serializer and undo the
    surgical cleanup (re-imposing `*` bullets, `***` breaks, escaping). A `setMarkdown` fallback
    covers the (source-mode-unlikely) case where the view is unreachable. Pass it to the toolbar.
- Docs: PRODUCT.md (capability, requirement, source-mode editing note, primary workflow, edge cases),
  ARCHITECTURE.md (`format.ts`/`CleanUpFormatting.tsx`/toolbar/`App.tsx` descriptions, a flow item, and
  a Clean Up Formatting workflow subsection), and this changelog entry.

## Verification

- `npm run build` (`tsc --noEmit` + vite) and `npm run test` (144 tests, including the 27 new `format`
  cases) pass.
- Browser preview (source mode, a messy fixture covering all transforms): the "Clean up formatting"
  button is absent in rich-text mode and present in source mode; clicking it rewrites the buffer to the
  exact surgical output — `-` markers, `1)` preserved, `_italic_` and `#Title` (a non-heading) left
  untouched, blank lines around `## Section`, an aligned right-justified table column, `---`, collapsed
  blanks, and a two-space hard break. Switching to rich-text and back confirmed MDXEditor parsed the
  cleaned source with no content loss (state stays synced through the direct CodeMirror edit), and the
  document is marked dirty. No console errors.
