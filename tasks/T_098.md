# T_098: Add an opt-in no-auth mode for the embedded MCP server

## Status

Done

## Summary

Add a second authentication mode to the embedded MCP server so users who only run trusted local AI clients can skip the bearer-token handshake. Default behavior stays unchanged (bearer-token required); selecting "No authentication" in the settings dialog turns off the `Authorization` check while keeping the loopback-only binding and the write confirmation dialog intact.

## Implementation Notes

- Extend `McpServerSettings` in `src/lib/settings.ts` with an `authMode` field of type `"bearer" | "none"`, default `"bearer"`. Add a sanitizer and update `createDefaultMcpServerSettings`, `sanitizeMcpServerSettings`, the default-settings unit tests, and the existing save-roundtrip test to include the new field.
- Update the settings dialog to expose the auth mode as a `<select>` with options "Bearer token (recommended)" and "No authentication". When the user picks "No authentication", show a short warning paragraph (e.g. `Any local process that can reach 127.0.0.1:{port} will be able to call MCP tools. Writes still require approval.`) and hide the bearer-token display, copy, and regenerate controls. When the user switches back to bearer, restore the existing controls, generating a new token if none exists.
- Forward `authMode` through `configureMcpServer` IPC and the renderer effect in `App.tsx` that already pushes MCP config to main.
- Extend the main-process `mcp:configure` handler and `electron/mcp-server.cjs` `configure()` to accept and store the `authMode`. Allow `enabled: true` with an empty bearer token only when `authMode === "none"`.
- In `electron/mcp-server.cjs` `handleRequest`, skip the bearer comparison when the active config has `authMode === "none"`. Keep the 127.0.0.1 check, the JSON parsing/error responses, and the write confirmation flow exactly as they are.
- Update the type definitions in `src/electron.d.ts` (`ConfigureMcpServerInput`) to include `authMode`.
- Do not change the write confirmation dialog or any tool implementations. The point of this task is auth-only.
