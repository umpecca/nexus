# T_046: Add Local, Remote, and Base64 Image Imports

## Status

Done

## Objective

Allow users to insert images into the editor from a local file path, a remote HTTP(S) URL, or an embedded base64 source.

## Scope

- Replace the generic MDXEditor image toolbar insertion button with a Nexus-owned shadcn-styled image import dialog.
- Keep MDXEditor's `imagePlugin` for Markdown image rendering, editing, import, and export behavior.
- Add Electron IPC for choosing a local image file and returning a `file:///` source.
- Add Electron IPC for choosing an image file and returning an embedded base64 data URL.
- Allow pasted base64/data URL values in the image import dialog.
- Update AI-DLC product, architecture, and changelog documentation.

## Acceptance Criteria

- Given the user chooses Local, when they select an image file, then Nexus inserts an image using a local file URL source.
- Given the user chooses Remote URL, when they enter an `http` or `https` image URL, then Nexus inserts an image using that URL.
- Given the user chooses Base64, when they choose an image file to embed, then Nexus inserts an image using a data URL.
- Given the user chooses Base64, when they paste a base64 or data URL value, then Nexus inserts an embedded image source.

## Implementation Notes

- Added `InsertImageImport` using MDXEditor's `insertImage$` publisher.
- Added `image:select-local` and `image:select-base64` main-process handlers.
- Returned local images as `file:///` URLs so Electron can render them reliably.
- Returned base64 imports as `data:{mime};base64,...` URLs.
