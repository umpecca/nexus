# T_081: Add resilient PDF print fallbacks for preview generation

## Status

Done

## Summary

Reduce repeated Print Preview failures by retrying PDF generation through progressively simpler Chromium print configurations when the primary CSS-sized print path fails.

## Implementation Notes

- Extracted hidden-window PDF printing into a shared helper that loads export HTML, renders Mermaid diagrams, waits for fonts and paint readiness, and calls `printToPDF`.
- Loaded hidden export windows from temporary HTML files instead of `data:` URLs, then cleaned those files up after rendering.
- Kept the primary CSS `@page` print path for matching paper size, orientation, and margins.
- Added a fallback path that regenerates export HTML without `@page` and calls Electron `printToPDF` with validated paper size, orientation, and pixel-based custom margins.
- Added a final default print fallback so preview/export can still produce a PDF when Chromium rejects configured print settings.
