# T_128: Embed a visual OpenAPI YAML editor

## Status

Done

## Summary

Adapt the MIT-licensed `waterrmalann/openapi-editor` into an offline Nexus editor for OpenAPI YAML
stored directly in Markdown as a ```` ```yaml openapi ```` fenced block. New blocks are inserted from
the WYSIWYG toolbar after the visual editor saves; existing blocks reopen in the same editor.

## Acceptance Criteria

- The Insert toolbar opens a modal visual OpenAPI editor and inserts a portable `yaml openapi` fence
  at the captured root or nested-editor caret only after Save.
- Rich-text mode shows a compact OpenAPI summary card with an Edit action; source mode and Markdown
  persistence retain the YAML fence unchanged.
- The offline editor supports API information, routes, responses, schemas, servers, tags, security
  schemes, YAML preview, undo/redo, pasted YAML, and local YAML/JSON import.
- Unknown OpenAPI fields and `x-*` extensions survive edits because the visual forms mutate the
  parsed source object directly; deleting or renaming an entity removes or moves its complete node.
- Invalid YAML or missing required OpenAPI structure blocks Save with a concrete error.
- New documents use OpenAPI 3.0.3; imported 3.x versions are retained and 3.1 shows a compatibility
  warning.
- Closing or canceling the modal leaves the Markdown document unchanged.
- The upstream MIT notice is retained, documentation is updated, and focused/full verification
  passes.

## Implementation Notes

- Use a separate Vite host page and a narrow Electron preload/IPC bridge, following the existing
  isoflow window pattern.
- Use the `yaml` package for parsing and normalized output. Comments and formatting may normalize,
  but data outside the visual editor's known fields must remain present.
- Keep ordinary YAML fences on the normal CodeMirror path by matching both language `yaml` and the
  `openapi` metadata token.
