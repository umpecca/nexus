# T_015-Shadcn MDXEditor Toolbar

**Task ID:** T_015

**Task Description:** Replace MDXEditor's built-in Kitchen Sink toolbar with a project-owned shadcn-styled toolbar while retaining the same editor command coverage.

**Status:** CLOSED

## Acceptance Criteria

1. The project includes local shadcn-style UI primitives needed by the toolbar.
2. The MDXEditor toolbar no longer renders `KitchenSinkToolbar`.
3. The replacement toolbar keeps the Kitchen Sink command set: undo/redo, text formatting, lists, block type, links, images, tables, thematic breaks, code blocks, Sandpack blocks, admonitions, frontmatter, and rich/diff/source controls.
4. The toolbar remains sticky and does not overlap the editable document area.

## Implementation Notes

- Added shadcn-style `Button` and `Separator` primitives plus a shared `cn` utility.
- Added `ShadcnMdxToolbar`, a project-owned toolbar composition that mirrors MDXEditor's Kitchen Sink toolbar using the same MDXEditor command components.
- Registered the custom toolbar through `toolbarPlugin` and applied local shadcn-style toolbar classes.
- Updated toolbar styling so MDXEditor command buttons, toggle groups, selects, and dropdowns match the local shadcn component treatment.

## Verification

Run `npm run build` and `npm run test`. Launch the app and verify the toolbar exposes the same editor controls as the previous Kitchen Sink toolbar and remains sticky while editing long documents.
