# T_113: Add Find & Replace to the in-editor find panel

## Status

Done

## Summary

Extend the existing find-only panel into a full find-and-replace experience, the single most-missed
editing affordance for working with docs. The panel keeps its current literal text search, match
count, and previous/next navigation, and gains a collapsible second row with a replacement field plus
replace-current and replace-all controls. A new Edit/Replace menu item (Ctrl/Cmd+H) opens the panel
with the replace row already expanded, while Edit/Find (Ctrl/Cmd+F) keeps opening the panel as before.
Replacement text is inserted literally (no regex group substitution), reusing the `replace` and
`replaceAll` helpers already exposed by MDXEditor's `searchPlugin`, so the heavy lifting of rewriting
the matched ranges and re-indexing the document stays in the plugin.

## Implementation Notes

- `src/components/editor/FindTextPanel.tsx`:
  - Add a `replaceRequest: number` counter prop (mirrors the existing `openRequest`) and pull
    `replace` and `replaceAll` from `useEditorSearch()` alongside the existing search controls.
  - Track `replacement` text and a `showReplace` boolean in component state; add a `replaceInputRef`
    and a `handledReplaceRequestRef` (matching the existing `handledOpenRequestRef` pattern).
  - Handle the replace request with a dedicated effect: when `replaceRequest` advances, expand the
    replace row, open the search, and focus/select the find input (so the user can refine the query
    first, the same as Find).
  - Restructure the markup into a toggle button plus a `.nexus-find-body` column that holds the find
    row and, when `showReplace`, a replace row. The toggle (chevron) shows/hides the replace row and
    moves focus to the appropriate input; `closePanel` clears the query and replacement and collapses
    the replace row.
  - Replace controls are disabled unless there is a non-empty query with at least one match; an empty
    replacement is allowed (deletes matches). Enter in the replace field replaces the current match;
    Escape closes the panel from either field.
- `src/styles.css`:
  - Add `.nexus-find-toggle` (28px ghost icon button on the panel's left edge), `.nexus-find-body`
    (flex column), and `.nexus-find-row` (flex row) rules; give `.nexus-find-input` `flex: 1 1 auto`
    so the replace input grows to align the replace controls' right edge with the find row's controls.
  - Fold `.nexus-find-toggle svg` into the existing 14px icon-sizing rule.
- `src/App.tsx`:
  - Add `pendingReplaceRequest` state and an `openReplacePanel` handler that bumps it; register the
    handler in both `menuHandlersRef` objects, add a `"replace"` case to `dispatchMenuAction`, handle
    Ctrl/Cmd+H in the existing `handleViewShortcut` keydown listener, and pass `replaceRequest` to
    `<FindTextPanel>`.
- `src/electron.d.ts`: add `"replace"` to the `NexusMenuAction` union.
- `electron/main.cjs`: add an Edit/Replace menu item (`CmdOrCtrl+H`) right after Edit/Find that sends
  the `"replace"` menu action.
- Docs: update `PRODUCT.md` (capabilities, requirements, primary workflow) and `ARCHITECTURE.md`
  (FindTextPanel component description, workflow #25, and the renamed Find and Replace Workflow
  section), and add this changelog entry.
