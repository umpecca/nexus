# T_142: Persist AI chats in user data

## Status

Done

## Summary

Persist AI-chat conversations locally for saved Markdown documents in the Nexus user-data directory.

## Acceptance Criteria

1. Saved documents restore their AI chat transcript and agent conversation after reopening the chat.
2. Untitled documents keep chat session-only and do not create history files.
3. User-data filenames are opaque and scoped to the OS profile plus canonical document path.
4. Clear Conversation deletes the saved history for the active document and clears the panel.
5. AI Provider settings provide a confirmed action to delete all saved chat history for the profile and
   clear the active panel.
6. API keys are not included in chat history files.
7. Automated tests cover history-path isolation and stored-data sanitization.

## Completion Notes

- Added an `ai-chats` user-data store with SHA-256 profile/document filenames and atomic writes.
- Persisted transcript display items and the agent conversation only for saved documents.
- Added load, save, delete-current, and delete-all IPC methods plus preload/type declarations.
- Routed Clear Conversation through deletion of the active document's history file.
- Added a confirmed Delete all saved chat history action in AI Provider settings.
