# T_080: Route PDF paper settings through CSS page sizing

## Status

Done

## Summary

Fix repeated Print Preview PDF generation failures by moving paper size, orientation, and margin settings into the rendered export document's print CSS before Electron generates the PDF.

## Implementation Notes

- Added Letter and A4 dimensions in inches for PDF print CSS generation.
- Injected a CSS `@page` rule with the selected page size, orientation, and margins into the PDF export HTML.
- Switched the shared PDF renderer to call `printToPDF` with `preferCSSPageSize` and `printBackground`.
- Passed the generated print CSS through the shared Markdown export renderer so print preview and PDF export use the same path.
