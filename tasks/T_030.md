# T_030: Prompt before opening over unsaved changes

**Task Description:** Make File/Open protect a dirty editor buffer before replacing the current document.

**Task Status:** Completed

**Task Status Details:** Superseded by T_031. File/Open still protects dirty buffers, but the dirty-buffer confirmation now appears only after the user selects a file in the native open dialog.

## Product Context

Opening a document replaces the active editor buffer. If the current buffer has unsaved changes, Nexus should behave like a traditional desktop app and ask whether the user wants to save, discard, or cancel before continuing.

## Acceptance Criteria

1. If the editor buffer is clean, File/Open opens the file picker normally.
2. If the editor buffer is dirty, File/Open prompts the user to Save, Don't Save, or Cancel.
3. Choosing Save saves the current buffer before the selected file replaces it.
4. Choosing Don't Save opens the selected file without saving the current buffer.
5. Choosing Cancel leaves the current buffer unchanged and does not replace it.
6. The app build and test suite continue to pass.

## Implementation Notes

- Reused `confirmDirtyBufferAction()` from `src/App.tsx` for File/Open after the native open dialog returns a selected file.
- Kept the existing native Electron unsaved-change message box.
- Reset the editor scroll snapshot after opening or creating a document so the replacement document starts at the top.

## Verification

Run `npm run build` and `npm run test`. Launch the app, type into a clean untitled document, choose File/Open, and verify canceling the native picker shows no save prompt. Select a file and verify Save, Don't Save, and Cancel protect the dirty buffer correctly before replacement.
