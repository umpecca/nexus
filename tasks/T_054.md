# T_054: Render admonitions correctly in exports

**Task Status:** Completed

**Task Description:** Render supported admonition directives as styled callout blocks in HTML and PDF exports instead of showing raw directive syntax.

## Context

Nexus uses MDXEditor's directive plugin for rich-text admonition editing, but native exports render Markdown through Marked. Marked does not understand `:::note`-style container directives by default, so exported PDFs showed the directive markers as plain text.

## Acceptance Criteria

1. `:::note`, `:::tip`, `:::info`, `:::caution`, and `:::danger` blocks render as callouts during PDF export.
2. The same supported admonition blocks render as callouts during HTML export.
3. Admonition body Markdown is rendered inside the callout.
4. Mermaid diagrams, images, tables, code blocks, and frontmatter handling continue to work.
5. Raw directive markers do not appear in exported output for supported admonitions.

## Implementation Notes

- Added an export-side directive scanner for supported admonition container blocks.
- Rendered admonition body content through the existing Marked export renderer before wrapping it in callout HTML.
- Added export CSS for note, tip, info, caution, and danger callout variants.
- Kept the renderer/editor directive behavior unchanged.

## Verification

Run `npm run build`, `npm run test`, and `git diff --check`. Manually export the built-in demo document to PDF and confirm the note and tip admonitions render as styled blocks without raw `:::` markers.
