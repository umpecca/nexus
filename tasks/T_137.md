# T_137: Add OpenCode Serve as an AI provider

## Status

Done

## Summary

Add an `opencode` AI provider backed by the `opencode serve` HTTP session API. The provider discovers
agents and connected models, supports optional HTTP Basic authentication, uses disposable sessions
for one-shot AI actions, and keeps one OpenCode session for each live Nexus chat conversation.
OpenCode executes its configured agent tools; Nexus displays that activity without routing those
calls through the Nexus MCP tool loop.

## Acceptance Criteria

1. OpenCode appears in AI Providers with server URL, username, optional password, agent, upstream
   provider, and model configuration.
2. Connection testing discovers server health, agents, connected providers, defaults, and models,
   while all discovered identifiers remain manually editable.
3. Selection actions, AI chat, and supported PDF/image imports use the OpenCode session/message API.
4. One-shot actions delete their OpenCode sessions; chat reuses a session until the conversation is
   cleared, its provider changes, its window closes, or Nexus exits.
5. OpenCode tool activity is visible in chat, configured tools run under OpenCode, and permission
   requests offer Reject, Allow once, and Always allow.
6. Basic-auth passwords never enter persisted renderer settings or ordinary AI request payloads and
   remain memory-only when operating-system encryption is unavailable.
7. Existing AI providers keep their current request and streaming behavior.
8. Tests, build, and Electron main-process syntax checks pass.

## Implementation Notes

- Added a direct CommonJS OpenCode adapter beside the existing provider adapter for request building,
  response/event parsing, image/file parts, discovery normalization, and tolerant SSE events.
- Kept OpenCode session ownership, Basic-auth passwords, permission decisions, aborts, and cleanup in
  the Electron main process. Secrets fall back to process memory when `safeStorage` is unavailable.
- Added an optional conversation ID and provider-tool activity events to the AI stream bridge. The
  renderer displays OpenCode's tools without invoking Nexus MCP tools.
- Added discovery-backed editable agent/provider/model fields, dedicated OpenCode sampling behavior,
  settings migration tests, adapter tests, and product/architecture/changelog documentation.
- Follow-up: replaced the provider and model free-text fields with dependent dropdowns populated from
  OpenCode discovery, including automatic loading for enabled configurations, explicit refresh, and
  a retained unavailable-value option for older saved settings.
- Follow-up: fixed PDF/image import path normalization so multi-file selections pass only the file
  name to Node's path resolver instead of leaking `Array.map`'s index and source-array arguments.
- Follow-up: made the native canvas dependency lazy so ordinary image and text-only PDF imports do
  not load it, and explicitly unpacked its JavaScript/native files for packaged PDF rasterization.
- Follow-up: replaced unpdf's deliberately unavailable serverless PDF.js canvas mock with a real
  document-level canvas factory, covering scanned-page rendering and embedded-image decoding.
- Follow-up: sanitized imported model output as non-executable Markdown data, escaping MDX JSX and
  expression delimiters outside code so mathematical less-than signs and LaTeX braces parse safely.
- Follow-up: instructed transcription models to emit Nexus `math` fences and normalized returned
  inline/display TeX into those fences, while neutralizing unsupported imported directive prefixes.
- Follow-up: retained standalone source images beside the model's description, preserving selected
  SVG files natively and avoiding duplicate insertion. For scanned PDFs, the vision model identifies
  normalized illustration bounds and Nexus inserts locally cropped PNGs instead of full-page images.
- Follow-up: added a small crop bleed, weighted toward the right edge, and matching model guidance
  so diagram labels, arrowheads, and border strokes are not clipped by tight vision bounds.
- Follow-up: added an Adjust crop toolbar action for ordinary raster images. Its dialog provides four
  draggable crop edges, a shaded live preview, Reset/Cancel/Apply controls, repeated adjustment from
  the original automatic crop during the editor session, portable PNG output, and normal Undo.
- Follow-up: reduced the crop selection border and visible edge grips while retaining narrow drag
  targets, making the exact retained boundary easier to inspect.
- Follow-up: changed the top/bottom crop handles from full-edge hit bars to compact centered grips,
  matching the reduced visual footprint of the left/right controls.
- Follow-up: increased those centered top/bottom grips to a still-compact 20-by-4-pixel pill with a
  dark outline so they remain visible against the white crop boundary and light image content.
- Follow-up: matched the left/right grips to the same pill design, rotated to 4-by-20 pixels with the
  same outline, centered hit area, and visual weight.
- Follow-up: added inline KaTeX as portable `` `math:...` `` code spans, with a priority rich-text
  visitor, toolbar insertion, export rendering, and import normalization that keeps sentence-level
  formulas inline while retaining fenced `math` blocks for display equations.
- Nexus does not start or manage the `opencode serve` process.
