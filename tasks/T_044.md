# T_044: Add GitHub Actions Desktop Builds

## Status

Done

## Objective

Build Nexus desktop artifacts for Windows and macOS automatically when changes are merged into `develop`.

## Scope

- Add Electron packaging scripts for Windows and macOS.
- Configure electron-builder output under `release/`.
- Add a GitHub Actions workflow triggered by pushes to `develop` and manual dispatch.
- Upload Windows and macOS build artifacts from the workflow run.
- Document the CI packaging behavior in the AI-DLC project files.

## Acceptance Criteria

- Given a change is pushed to `develop`, when GitHub Actions runs, then it builds the Windows desktop package on a Windows runner.
- Given a change is pushed to `develop`, when GitHub Actions runs, then it builds the macOS desktop package on a macOS runner.
- Given a desktop package is created, when the workflow completes, then the generated artifacts are uploaded to the workflow run.
- Given a developer wants to package locally, when they build the renderer and run the platform package script, then electron-builder writes artifacts to `release/`.

## Implementation Notes

- Added `dist:win` and `dist:mac` package scripts for packaging an already-built renderer.
- Added electron-builder metadata for Nexus, x64 Windows NSIS/portable output, and macOS DMG/ZIP output.
- Added `.github/workflows/build-desktop.yml` with a Windows/macOS matrix.
- Disabled code-signing identity auto-discovery in CI so unsigned builds can complete without release credentials.
