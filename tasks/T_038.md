# T_038: Add manual refresh from disk

**Task Description:** Add a manual Refresh action in the editor toolbar and native Edit menu that reloads the current opened file from disk.

**Task Status:** Complete

**Task Status Details:** Added Edit/Refresh menu forwarding, a toolbar refresh button, renderer refresh logic, and conflict prompt copy for manual reloads.

## Context

Nexus can already detect external file changes, but users also need a traditional desktop command to manually reload the current file from disk. Refresh should be quiet when no unsaved editor content is at risk and protective when disk content differs from a dirty buffer.

## Acceptance Criteria

1. Edit/Refresh is available from the native app menu.
2. A refresh button is available from the editor toolbar.
3. Untitled documents do not attempt to refresh from disk.
4. Clean buffers reload from disk without prompting.
5. Dirty buffers reload without prompting when the current editor contents match disk.
6. Dirty buffers prompt before replacing local edits when disk content differs.
7. Deleted or moved files show the existing missing-file prompt and keep the buffer open.

## Implementation Notes

- Added `refresh` to the renderer menu action contract.
- Reused `readWatchedMarkdownFile()` for manual disk reads and watcher signature updates.
- Passed refresh props into `ShadcnMdxToolbar` and used a lucide refresh icon.
- Reused `FileChangedDialog` for manual refresh conflicts and missing-file feedback.
