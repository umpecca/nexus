# T_111: Make the outline sidebar horizontally resizable

## Status

Done

## Summary

Let the user drag the right edge of the outline sidebar to change its width, so long heading text that would otherwise be clipped can be read in full. The chosen width is clamped to a sensible range, never crowds out the editor, persists per OS profile alongside the other editor settings, and can also be adjusted from the keyboard for accessibility.

## Implementation Notes

- Add an `outlineWidthPixels` number to the settings model in `src/lib/settings.ts`.
  - Add `DEFAULT_OUTLINE_WIDTH_PIXELS` (256, matching the previous fixed `16rem`), `OUTLINE_WIDTH_MIN_PIXELS` (180), and `OUTLINE_WIDTH_MAX_PIXELS` (560).
  - Sanitize on load by rounding and clamping to `[min, max]`, falling back to the default for non-numeric values; store it inside the existing `nexus:settings:v1:{profileName}` object without adding a new key.
- Extend `src/lib/settings.test.ts` with cases for the new field's default, valid load (with rounding), range clamping, and non-numeric fallback, and update the existing full-shape assertions.
- Drive the sidebar width from a `--outline-width` CSS variable in `src/styles.css`:
  - Replace the hardcoded `16rem` in both `.editor-surface > .nexus-outline` (width) and the `.editor-surface-with-outline .mdxeditor-rich-text-editor` left padding with `var(--outline-width, 16rem)` so the panel and the editor's reserved space stay in lockstep.
  - Add a `.nexus-outline-resizer` drag handle pinned to the panel's right edge (col-resize cursor, accent hairline on hover/focus) and inset the scrolling list (`margin-right`) so the handle never overlaps the scrollbar.
  - Add a `body.nexus-resizing-col` rule that forces the resize cursor and suppresses text selection while a drag is in progress.
- Update `src/components/editor/OutlineSidebar.tsx`:
  - Accept `width` and `onResize(width)` props.
  - Render a `role="separator"` handle with `aria-orientation="vertical"` and `aria-valuemin/max/now`, focusable via `tabIndex`.
  - On pointer drag, update the live width by writing `--outline-width` directly on the editor surface (no per-move re-render, so dragging stays smooth) and commit the final value via `onResize` on pointer up.
  - Clamp the dragged width to `[min, max]` and additionally to the container width minus a minimum editor width so the editor always keeps room.
  - Support `ArrowLeft`/`ArrowRight` to nudge the width in fixed steps.
- Wire it in `src/App.tsx`:
  - Add `--outline-width` (from `settings.outlineWidthPixels`) to the editor surface inline style.
  - Pass `width` and an `onResize` that updates `outlineWidthPixels` through the existing `setSettings` flow so the width persists per profile.

## Verification

- Manual test: enable the outline (View / Show Outline), drag the right edge to widen and narrow the panel, and confirm the editor content reflows to match and long headings become readable.
- Manual test: confirm the width cannot be dragged below the minimum, above the maximum, or wide enough to crowd out the editor on a narrow window.
- Manual test: focus the resize handle and confirm Arrow Left/Right adjust the width.
