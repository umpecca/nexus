# T_041: Fix close cleanup after external file reloads

**Task Description:** Prevent intermittent Electron main-process errors when closing a window after external file reload activity.

**Task Status:** Complete

**Task Status Details:** Captured each window's `webContents.id` during creation and reused that stable ID during the `closed` event to clean up file watchers and pending initial-file state.

## Context

After a file was externally edited and reloaded, closing Nexus could sometimes show a main-process JavaScript error: `TypeError: Object has been destroyed` at the watcher cleanup line. The close handler was reading `window.webContents.id` after Electron had already destroyed the webContents object.

## Acceptance Criteria

1. Window close cleanup must not read `window.webContents` after the window has closed.
2. File watchers are still stopped for the closing window.
3. Pending initial file payloads are removed when their window closes.
4. Closing after an external reload does not show a main-process JavaScript error.

## Implementation Notes

- Captured `webContentsId` immediately after creating the `BrowserWindow`.
- Reused the captured ID for pending initial file registration.
- Reused the captured ID for `stopFileWatcher()` inside the `closed` event.
- Deleted pending initial file state during `closed` cleanup.
- Guarded watcher event window lookup against destroyed `BrowserWindow.webContents` objects.

## Verification

Run `node --check electron/main.cjs`, `node --check electron/preload.cjs`, `npm run test`, and `npm run build`. Manually verify opening a watched file, reloading after an external edit, and closing the window.
