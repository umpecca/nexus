# T_034: Keep scroll-only interactions from marking opened files dirty

**Task Status:** Completed

**Task Description:** Fix dirty-buffer detection so opening a file and scrolling through it does not mark the document as modified when the editor only normalizes line endings internally.

## Context

MDXEditor can emit Markdown after non-edit interactions, and that emitted Markdown may use normalized line endings. Files opened from disk can contain Windows-style CRLF line endings, so a raw string comparison can make a clean document appear dirty after scrolling or focus-related editor updates.

## Acceptance Criteria

1. Opening a file and scrolling without editing must keep the window title clean.
2. File/New, File/Open, and close prompts must not appear for a buffer changed only by line-ending normalization.
3. Real content edits must still mark the document dirty and trigger the existing prompts.
4. The saved Markdown content path must continue to use the editor's current Markdown buffer.

## Implementation Notes

- Added Markdown dirty-comparison helpers that normalize CRLF/CR line endings to LF before comparison.
- Updated renderer title and dirty-confirmation checks to use the normalized comparison.
- Added tests covering line-ending-only equivalence and meaningful content changes.

## Verification

Run `npm run build` and `npm run test`. Launch the app, open a CRLF Markdown file, scroll through it without editing, and confirm the title does not gain an asterisk and close/open/new do not prompt for unsaved changes.
