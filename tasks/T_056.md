# T_056: Add paper-size editing and matching PDF export

**Task Status:** Completed

**Task Description:** Add a first-pass print-layout experiment so rich-text editing uses a selected Letter or A4 paper width and PDF export uses the same selected paper size.

## Context

Nexus exports to PDF, but the editor previously used an edge-to-edge writing surface while PDF export used a fixed A4 page. Users who print often need the on-screen writing width to better reflect the final paper output so images, tables, and other elements do not feel unexpectedly large or spill past the page.

## Acceptance Criteria

1. Settings allows choosing Letter or A4 paper size.
2. The selected paper size is stored per OS profile with the existing editor settings.
3. Rich-text editing shows a centered paper-width writing surface with normal one-inch margins.
4. PDF export uses the selected paper size.
5. Existing font settings, HTML export, source mode, diff mode, image preview, Mermaid export, and admonition export continue to work.

## Implementation Notes

- Extended the local settings model with a `pageSize` value that defaults to Letter and migrates existing font-only settings safely.
- Added a Paper size select to the settings dialog.
- Applied paper width, height, and margin CSS variables from the app shell to the rich-text editor surface.
- Moved the paper frame styling onto MDXEditor's root editable wrapper so the empty-document placeholder does not render as a second page on first launch.
- Kept true pagination and custom margins out of scope for this first experiment.
- Passed the selected paper size through preload IPC and validated it in the Electron main process before calling `printToPDF`.
- Used one-inch PDF export margins so Electron's `printToPDF` margin validation accepts Letter and A4 output.

## Verification

Run `npm run build`, `npm run test`, and `git diff --check`. Manually switch between Letter and A4 in Settings, confirm the rich-text page width changes, then export a PDF and confirm it uses the selected paper size without changing the active document dirty state.
