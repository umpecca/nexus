# T_129: Add a Swagger-style OpenAPI document preview

## Status

Done

## Summary

Expand embedded `yaml openapi` blocks in rich-text mode into a useful, read-only API reference while
preserving the portable YAML fence in Markdown and keeping the existing visual editing workflow.

## Acceptance Criteria

- The compact OpenAPI card can expand and collapse an inline API reference without opening the editor.
- The reference shows the API description, servers, and operations grouped by their first tag.
- Each operation shows its HTTP method, path, summary/description, parameters, request body, responses,
  and referenced schema details where available.
- HTTP methods use familiar Swagger-style colors and the preview follows Nexus light/dark themes.
- The existing Edit visually action remains available and editing still updates only the source YAML.
- Malformed OpenAPI YAML retains the current actionable error card.
- The preview is read-only and offline; it does not send API requests or load remote assets.
- Tests cover operation extraction, grouping, parameter/request/response summaries, schema references,
  and rendering behavior.
- PRODUCT, ARCHITECTURE, and CHANGELOG describe the richer document preview.

## Implementation Notes

- Build the preview from the existing parsed OpenAPI object instead of adding Swagger UI and its
  browser/network-oriented dependency surface.
- Keep all preview output as normal React/HTML so Electron export and theme behavior remain under
  Nexus control.

## Verification

- `npm run test` passes (50 files, 521 tests).
- `npm run lint` passes.
- `npm run build` passes and retains the separate OpenAPI editor Vite entry.
- Focused tests cover tagged reference extraction, path/operation parameter merging, local schema
  references, request/response media, rendered operation details, and empty specifications.
- A local browser smoke test rendered a representative cash-price API in rich-text mode and verified
  the method-colored expanded operation, server/security/parameter/body/response details, and the
  Hide reference / Show reference interaction. No API request was issued.
