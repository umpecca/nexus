# T_039: Add MDXEditor diff review workflow

**Task Description:** Add first-class diff review support using MDXEditor's diff mode for external file conflicts and previous-version comparison.

**Task Status:** Complete

**Task Status Details:** Added renderer diff baselines, programmatic diff-mode switching, a conflict-dialog Review Diff action, previous-version tracking, and an Edit menu command for comparing with the prior version.

## Context

Nexus already watches opened files and prompts when disk changes would conflict with a dirty editor buffer. Users also need a way to inspect those differences before deciding whether to reload, keep editing, or save a copy. MDXEditor already provides diff mode, so Nexus should reuse it instead of building a separate review surface.

## Acceptance Criteria

1. MDXEditor is configured with `diffSourcePlugin` using a renderer-controlled diff baseline.
2. Dirty external file change conflicts offer a Review Diff action.
3. Review Diff compares the current editor buffer against the changed disk file without replacing the buffer.
4. Manual refresh dirty conflicts can also enter diff review against the disk version.
5. Successful saves preserve the saved Markdown from before the latest save as a previous-version baseline.
6. Edit/Compare with Previous Version switches MDXEditor into diff mode using the previous-version baseline.
7. Compare with Previous Version is a no-op when no previous baseline exists.
8. Reloading from disk still replaces the buffer and clears the dirty marker.

## Implementation Notes

- Added `diffMarkdown`, `previousVersionMarkdown`, and `pendingDiffViewRequest` renderer state.
- Added a small toolbar-scoped controller that publishes MDXEditor's `viewMode$` to switch into diff mode after a baseline is selected.
- Extended the file-change dialog with a shadcn-styled Review Diff button for dirty changed-file prompts.
- Stored changed disk contents on external-change and refresh prompts so review can happen without mutating the current buffer.
- Added `comparePreviousVersion` to the Electron menu action contract.
