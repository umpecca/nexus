# T_051: Render Mermaid diagrams in HTML and PDF exports

**Task Status:** Completed

**Task Description:** Render standard fenced Mermaid code blocks as static diagrams when exporting Markdown to HTML or PDF.

## Context

Nexus already renders Mermaid diagrams in rich text mode, but the native export workflow renders Markdown separately through Marked in the Electron main process. That export path needs its own Mermaid handling so documents with diagrams produce useful exported files.

## Acceptance Criteria

1. Fenced `mermaid` code blocks render as diagrams in exported PDF files.
2. Fenced `mermaid` code blocks render as static SVG diagrams in exported HTML files.
3. Invalid Mermaid syntax exports as a compact inline error instead of failing the whole export.
4. Non-Mermaid code blocks keep the normal Markdown export rendering.
5. Existing relative local image export behavior remains unchanged.

## Implementation Notes

- Added a Marked code renderer override that emits export-only Mermaid placeholders for fenced `mermaid` blocks.
- Added hidden BrowserWindow rendering for Mermaid placeholders using Mermaid's bundled browser script with strict security mode and the default theme.
- Updated PDF export to render Mermaid placeholders before calling `printToPDF`.
- Updated HTML export to serialize the hidden export window after Mermaid rendering so the saved file contains static SVG diagrams.
- Added export CSS for centered, page-width-constrained Mermaid SVGs and compact inline Mermaid errors.

## Verification

Run `npm run build`, `npm run test`, and `git diff --check`. Manually export a document containing a Mermaid diagram to HTML and PDF and confirm the diagram is rendered in both files.
