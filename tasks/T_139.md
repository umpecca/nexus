# T_139: Add authoritative print preview and retire approximate page guides

## Status

Done

## Summary

Add a print-preview workflow that shows Chromium's actual paginated PDF output. Keep Paper View as a
continuous editing surface and remove the experimental page-guide overlay because the editor and
export renderer lay out content differently and the guides could therefore mislead document authors.

## Acceptance Criteria

1. Paper View remains a continuous editor and exposes no page-guide toggle or pagination overlay.
2. The File menu exposes **Print Preview**, which renders the current unsaved Markdown using the
   same rich export renderer and page settings as **Export as PDF**.
3. Preview generation reuses the current direct Chromium `printToPDF` path and does not restore the
   removed retry, temporary-file, or text-fallback pipelines.
4. The preview displays the generated PDF in-app, can be refreshed after document changes, can save
   the displayed snapshot as a PDF, and can be closed without modifying the document.
5. Preview loading and rendering errors are shown in the app and do not leave hidden export windows
   or stale object URLs behind.
6. Print CSS keeps headings with following content, preserves paragraph/list widows and orphans, and
   avoids splitting exported admonitions, GitHub alerts, and other common blocks when they fit.
7. Existing rich-text, source, diff, HTML/Word/PDF export, and Markdown persistence behavior remains
   unchanged.
8. Automated tests cover menu routing, shared PDF rendering, fragmentation selectors, and preview
   lifecycle. Tests, lint, build, and Electron syntax checks pass.

## Implementation Notes

- Extract the working PDF rendering operation in `electron/main.cjs` only as far as needed for both
  Export as PDF and Print Preview to call the same direct implementation.
- Keep preview state and UI in the renderer. Revoke replaced or closed preview object URLs and destroy
  the one-off hidden render window in a `finally` block.
- Do not split Lexical/MDXEditor content into page containers. Print Preview is the sole pagination
  authority.
- Target the export renderer's emitted `.nexus-export-admonition` class for print fragmentation so
  both directives and GitHub alerts receive the intended rule.
