# T_084: Add a text-first fallback PDF renderer

## Status

Done

## Summary

Ensure Print Preview and Export as PDF still produce a valid PDF when every Electron `printToPDF` attempt fails with a native printing error.

## Implementation Notes

- Kept the existing rich Chromium PDF renderer as the primary export path.
- Added a built-in text-first PDF serializer as the final fallback after all native print attempts fail.
- Preserved selected page size, orientation, margins, and base font size in the fallback output.
- Included a short visible note in fallback PDFs explaining that Electron's native PDF printer failed.
- Avoided adding a new dependency so packaged builds keep the same install footprint.

## Verification

- Run `node --check electron/main.cjs`.
- Run `npm run test`.
- Run `npm run build`.
- Verify File/Print Preview no longer reports `Printing failed` when native PDF printing is unavailable.
