# T_121: ngrok stop/restart controls and an MCP landing page

## Status

Done

## Summary

Two follow-ups to the MCP verification work (T_120):

- **ngrok stop/restart buttons.** The MCP settings section gains "Restart tunnel" and "Stop tunnel"
  buttons that operate the running ngrok agent directly, separate from the enable toggle. Restart
  recovers a wedged tunnel or refreshes a random URL; Stop tears it down immediately.
- **Unauthenticated landing page + its URL in Preferences.** The MCP server serves a simple HTML page
  at `/` that just says the server is reachable ("Testing"), so a user can open the local URL or the
  public ngrok URL in a browser to confirm the path works (a browser GET cannot send the bearer
  token, so this page is intentionally public and exposes no document content). The settings dialog
  shows the local "Test page" URL and, when the tunnel is connected, the "Public test page" URL.

## Implementation Notes

- `electron/mcp-server.cjs`: add a static `LANDING_PAGE_HTML` and a `sendHtml` helper; in
  `handleRequest`, serve `GET /` (and `/index.html`) with the landing page before the `/mcp` check, no
  auth (a non-GET gets 405). Added `GET` to the OPTIONS allow-methods earlier in T_120 already covers
  preflight.
- `electron/main.cjs`: add `ipcMain.handle("mcp:stop-ngrok", ...)` (calls `ngrokTunnel.stopTunnel()`)
  and `ipcMain.handle("mcp:restart-ngrok", config)` (stops then `ensureTunnel` with the server's live
  listening port and the config's domain/command when the server is listening and ngrok is enabled).
  Helpers `ngrokParamsFromConfig` (mirrors the `mcp:configure` resolution of domain + explicit-path
  command) and `currentNgrokStatus` (shapes the `getTunnelState()` result like the configure response).
  Stopping first is required because `ensureTunnel` no-ops when already connected to the same target.
- `electron/preload.cjs`: expose `stopMcpNgrok()` and `restartMcpNgrok(config)`.
- `src/electron.d.ts`: add `stopMcpNgrok()` / `restartMcpNgrok(config)` returning `McpNgrokStatus`.
- `src/App.tsx`: pass `onStopNgrok` / `onRestartNgrok` to `SettingsDialog`; each calls the bridge
  (restart builds the config from `settings.mcpServer`) and updates `mcpNgrokStatus` from the result.
- `src/components/settings/SettingsDialog.tsx`: add the `onStopNgrok` / `onRestartNgrok` props, an
  `ngrokBusy` state with `handleStopNgrok` / `handleRestartNgrok`, derived `mcpLandingUrl` /
  `ngrokPublicLandingUrl`, a local "Test page" URL field (when the server is enabled), the
  Restart/Stop buttons in the ngrok subsection (Stop disabled when not connected), and a "Public test
  page" URL field (when the tunnel is connected).

## Verification

- `npm run build` (`tsc --noEmit` + vite) and `npm run test` pass (272 tests).
- `electron/mcpServer.health.test.ts`: added cases asserting `GET /` returns 200 `text/html`
  containing "testing" without a token even while bearer auth is on, and that a non-GET to `/` is 405.
- `node --check` on the edited `.cjs` files.
- Manual (desktop): open the local Test page URL in a browser → "Testing" page. Enable ngrok, open the
  Public test page URL → same page over the tunnel. Click "Restart tunnel" → the public URL refreshes
  and reconnects; "Stop tunnel" → the tunnel disconnects and the status/URLs clear.
