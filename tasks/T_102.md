# T_102: Fix the Marked renderer `this.parser` crash in the Word export pipeline

## Status

Done

## Summary

File / Export as Word Document throws `Cannot read properties of undefined (reading 'parse')` (followed by Marked's "please report to markedjs" message) whenever the document contains a blockquote, table, or any block whose default renderer walks back into nested tokens. Root cause: in `electron/main.cjs` `renderMarkdownExportHtml` (added in T_100), the captured default renderers are saved with `renderer.blockquote.bind(renderer)` and the override arrow functions then invoke those bound defaults. Marked v16's `use({ renderer })` actually merges our override methods onto its own internal renderer instance and only sets `parser` on that internal renderer — never on the local `renderer` we own. The bound defaults force `this === ourLocalRenderer`, so when the default `blockquote`/`table`/`tablecell` implementation runs `this.parser.parseInline(...)` it dereferences `undefined`.

Fix the four overrides so the captured defaults are *not* `.bind()`-ed and the overrides themselves are function expressions that pass `this` straight through with `.call(this, token)`. This restores Marked's normal call convention and lets `this.parser` resolve against Marked's real renderer where `parser` is set.

The bug only manifests in DOCX export because that is the only caller that passes `inlineStyles: true`, which is the only branch that installs the `blockquote`/`table`/`tablecell` overrides. HTML and PDF exports never hit the broken code path. The TurboDocx swap in T_101 did not introduce or change this bug; the crash happens during Marked parsing, before any HTML reaches `@turbodocx/html-to-docx`.

## Implementation Notes

- In `electron/main.cjs` `renderMarkdownExportHtml`:
  - Replace the four `.bind(renderer)` captures with bare references:
    ```js
    const defaultCodeRenderer = renderer.code;
    const defaultBlockquoteRenderer = renderer.blockquote;
    const defaultTableRenderer = renderer.table;
    const defaultTablecellRenderer = renderer.tablecell;
    ```
  - Convert each override that calls a captured default from an arrow function to a `function` expression so `this` flows in from Marked's wrapped invocation:
    ```js
    renderer.code = function (token) {
      if (!isMermaidFence(getCodeTokenLanguage(token))) {
        const rendered = defaultCodeRenderer.call(this, token);
        // ... unchanged inline-style replacement ...
      }
      // ... unchanged mermaid branch ...
    };
    ```
    The same shape applies to `renderer.blockquote`, `renderer.table`, and `renderer.tablecell` inside the `if (inlineStyles)` block — each captured default is invoked via `.call(this, token)`.
- `renderer.image` does not use `this` and can stay as an arrow function.
- `createMarkedHighlightExtension` already uses the method-shorthand `renderer(token) { this.parser.parseInline(...) }` form, which is correct; no change needed.
- `renderMarkdownAdmonitions` does not interact with Marked renderer overrides directly (it splits Markdown on admonition fences and re-invokes `parseMarkdown`), so no change there either.
- This is a single-file change in `electron/main.cjs`. Do not change behavior elsewhere; HTML / PDF exports must remain byte-identical, and the DOCX export should once again produce the same output it was supposed to after T_100.
