# T_059: Add a saved paper/plain editor view toggle

**Task Status:** Completed

**Task Description:** Add a profile-scoped toolbar toggle that lets rich-text editing switch between the print-like paper surface and a plain words-first surface.

## Context

Nexus now supports paper size, margins, and base font sizing for print-oriented editing, but users do not always want to see the simulated page while writing. A quick toggle should let them hide the paper sheet without changing export settings.

## Acceptance Criteria

1. The toolbar exposes a Paper view toggle near the existing view mode controls.
2. The selected paper/plain view is stored per OS profile.
3. Paper view remains the default for existing and new users.
4. Plain rich-text view removes the sheet, shadow, fixed page size, page height, and page margins.
5. Source mode, diff mode, HTML export, and PDF export continue to work with existing settings.

## Implementation Notes

- Extended profile settings with `paperViewEnabled`, defaulting to `true`.
- Passed the paper view state from the app shell into the project-owned MDXEditor toolbar.
- Added a toolbar toggle button with pressed state and tooltip text.
- Split rich-text surface styling into paper and plain editor surface classes.
- Kept PDF and HTML export behavior unchanged.

## Verification

Run `npm run test`, `npm run build`, and `git diff --check`. Manually toggle paper view on and off in rich text, switch between rich text/source/diff, and confirm the saved preference is restored on reload.
