# T_095: Add MCP server settings and preferences UI

## Status

Done

## Summary

Lay the foundation for the optional embedded MCP server: persist per-profile MCP settings, expose them in the settings dialog, and route a configure IPC call to the Electron main process. No HTTP server is started yet; this task is the off-by-default scaffold the next two tasks depend on.

## Implementation Notes

- Extend `UserSettings` in `src/lib/settings.ts` with an `mcpServer` group containing `enabled` (boolean, default `false`), `port` (number, default `39125`, clamped to `1024..65535`), and `bearerToken` (string, default empty). Add sanitizers, a default factory, and load/save coverage so existing stored settings continue to load.
- Generate the bearer token lazily when the user first turns the server on: if `enabled` flips to `true` while `bearerToken` is empty, the renderer fills it with `crypto.randomUUID()` (or a 32-character random hex string when `crypto.randomUUID` is unavailable). Allow the user to regenerate the token from the settings dialog.
- Add a new "MCP server (experimental)" section to `src/components/settings/SettingsDialog.tsx` with: enable toggle, port input (`number`, min 1024, max 65535), token display with copy button and Regenerate button, and a read-only connection URL field that mirrors `http://127.0.0.1:{port}/mcp` when enabled. Keep the section visually distinct (helper text explaining what it does and that writes still require user confirmation).
- Plumb new props through `SettingsDialog` and into `src/App.tsx` so toggling the controls updates `settings.mcpServer` state and triggers `saveSettings`.
- Add a `nexus.configureMcpServer(config)` preload method and matching `mcp:configure` IPC handler stub in `electron/main.cjs`. The handler should accept `{ enabled, port, bearerToken }`, store the latest configuration in a module-scoped variable, and return `{ ok: true }`. No HTTP listener is created in this task.
- From `App.tsx`, call `configureMcpServer` whenever `settings.mcpServer` changes (debounced or guarded against initial undefined state) so the main process always knows the current config.
- Add a `MCP_SERVER_DEFAULT_PORT`, `MCP_SERVER_MIN_PORT`, and `MCP_SERVER_MAX_PORT` constant set, plus `sanitizeMcpServerPort` and `sanitizeMcpServerSettings` helpers in `src/lib/settings.ts`.
- Do not start any HTTP listener. Do not register any window. Do not expose tools. The follow-up tasks (T_096, T_097) add server lifecycle, read tools, and the write confirmation flow.
