# T_036: Render Mermaid diagrams in rich text mode

**Task Status:** Completed

**Task Description:** Render standard Mermaid fenced code blocks as diagrams in rich text mode while keeping the raw Mermaid fence editable in source and diff modes.

## Context

Nexus already uses MDXEditor code block descriptors for specialized rich text behavior. Mermaid diagrams should use the same extension point so `mermaid` fences remain portable Markdown and source/diff mode can continue showing raw editable text.

## Acceptance Criteria

1. Fenced `mermaid` code blocks render as diagrams in rich text mode.
2. Mermaid diagrams are not editable in rich text mode.
3. Source and diff modes show the raw Mermaid fence as editable Markdown text.
4. Invalid Mermaid syntax shows an inline non-editable error in rich text mode.
5. The toolbar provides an Insert Mermaid diagram action with a starter diagram.
6. Existing local JavaScript runner and normal code block behavior remains unchanged.

## Implementation Notes

- Added Mermaid as a runtime dependency.
- Added a Mermaid render helper that initializes Mermaid with the default light theme and strict security mode.
- Added a high-priority MDXEditor code block descriptor for `mermaid` language blocks.
- Added a toolbar button that inserts a starter Mermaid flowchart fence.

## Verification

Run `npm run test` and `npm run build`. Launch the app, insert a Mermaid diagram, confirm it renders in rich text mode, switch to source mode to edit the fence, then switch back and confirm the preview updates.
