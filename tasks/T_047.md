# T_047: Resolve Relative Local Image Previews

## Goal

Render Markdown image sources like `radio.jpg` relative to the folder containing the opened Markdown file when the editor is in rich text preview mode.

## Scope

- Keep the Markdown source unchanged.
- Resolve only preview-time local image sources through the Electron main process.
- Leave remote URLs, data URLs, blob URLs, file URLs, and untitled-document relative paths unchanged.
- Convert absolute local paths to file URLs for preview consistency.

## Implementation Notes

- Added an Electron IPC handler for image preview resolution.
- Exposed the resolver through preload as `window.nexus.resolveImagePreview`.
- Registered an MDXEditor `imagePlugin` preview handler in the renderer that delegates to Electron with the current document path.

## Acceptance Criteria

- A Markdown file at `C:\docs\note.md` containing `![radio](radio.jpg)` previews `C:\docs\radio.jpg` in rich text mode.
- Source and diff modes continue to show the raw relative Markdown path.
- Existing HTTP(S), data, blob, and file URL images keep rendering without path rewriting.
