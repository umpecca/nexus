# T_032: Add multiple editor windows

**Task Description:** Allow Nexus to open multiple editor windows so multiple documents can be open at the same time.

**Task Status:** Completed

**Task Status Details:** Added File/New Window, made Electron window creation reusable, kept document menu actions focused-window scoped, and updated the quit flow to prompt dirty windows one at a time before exiting.

## Product Context

Nexus should behave like a traditional desktop editor where multiple documents can be open simultaneously. A window-based model is the smallest fit for the current app because each renderer already owns its own document state, title, dirty tracking, and editor instance.

## Acceptance Criteria

1. File/New Window opens another blank untitled editor window.
2. Multiple editor windows can remain open at the same time.
3. File/New, File/Open, File/Save, File/Save As, Settings, and Help actions target the currently focused window where applicable.
4. Closing a dirty window prompts Save, Don't Save, or Cancel for that window.
5. Quitting the app prompts each dirty window before exiting.
6. Canceling any dirty-buffer prompt during app quit stops the quit and leaves remaining windows open.
7. The app build and test suite continue to pass.

## Implementation Notes

- Added File/New Window with `CmdOrCtrl+Shift+N`.
- Returned the created BrowserWindow from `createWindow()`.
- Replaced the single-window quit flag with a multi-window quit coordinator.
- Kept per-window close state in the existing `closeStates` map.
- Continued sending document menu actions to `BrowserWindow.getFocusedWindow()`.

## Verification

Run `npm run build` and `npm run test`. Launch the app, choose File/New Window, and confirm two independent blank editor windows are open. Edit both, close one window, and confirm only that window prompts. Choose File/Exit with multiple dirty windows and confirm each dirty window is prompted before quitting, with Cancel stopping the quit flow.
