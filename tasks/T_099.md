# T_099: Add Microsoft Word (.docx) export from the File menu

## Status

Done

## Summary

Add a third export target alongside HTML and PDF: File/Export as Word Document. Reuse the existing Marked-rendered export HTML (with admonition callouts, code block styling, and Mermaid SVG rendering) so the Word output preserves as much rich formatting as possible. Rasterize Mermaid SVGs to PNG inside the hidden export window so Microsoft Word reliably displays the diagrams, then convert the HTML to a `.docx` buffer through `html-to-docx`.

## Implementation Notes

- Add `html-to-docx` to `package.json` dependencies. The package exports a CommonJS function compatible with the Electron main process. Verify with `npm install html-to-docx` and `node -e "require('html-to-docx')"`.
- In `electron/main.cjs`:
  - Add a `docxFilters` constant alongside the existing `htmlFilters` / `pdfFilters` (`.docx`).
  - Add a `renderExportSvgsAsImages(webContents)` helper that runs inside the hidden export window. For every `<svg>` element in the rendered body, serialize the SVG to a string, draw it onto an offscreen `HTMLCanvasElement` at the SVG's intrinsic width/height (falling back to bounding-rect size), convert the canvas to a `data:image/png;base64,...` URL via `toDataURL("image/png")`, then replace the original `<svg>` node with an `<img>` carrying that data URL plus matching width/height attributes. Wrap the work in a Promise so the main process can await it. If a single SVG fails to rasterize, leave the SVG in place and continue with the others.
  - Add an `extractExportWindowHtml(webContents)` helper that returns `document.documentElement.outerHTML` from the export window.
  - Add a new `ipcMain.handle("file:export-docx", ...)` handler that mirrors the PDF handler: render Markdown to export HTML with `excludeFrontmatter: true`, open a save dialog with `docxFilters`, create the hidden export window, render Mermaid SVGs, rasterize them to PNGs, extract the final HTML, call `htmlToDocx(html, null, { orientation, margins, pageSize })` (translating Letter/A4 to twip dimensions for the `pageSize` option and inches to twips for margins), and write the resulting Buffer with `fs.writeFile`. Use a `try/finally` to destroy the export window. On error, route through `showExportError(event, "Word", error)`.
- In `electron/preload.cjs`, add `exportMarkdownAsDocx(currentPath, markdown, options)` that invokes `file:export-docx` with the same payload shape as PDF.
- In `src/electron.d.ts`, add `ExportMarkdownDocxOptions` (identical shape to `ExportMarkdownPdfOptions`) and the `exportMarkdownAsDocx(...)` method.
- In `src/App.tsx`:
  - Add `exportDocumentAsDocx` mirroring `exportDocumentAsPdf`, passing the same font/page settings.
  - Add `exportDocx` to the menu handlers ref and to the menu-action subscriber switch.
- Extend the native File menu in `buildMenu()` so there is an `Export as Word Document...` item directly below `Export as PDF...`.
- Extend `NexusMenuAction` in `src/electron.d.ts` with `"exportDocx"`.
- Twip conversion: 1 inch = 1440 twips. Letter portrait is 12240 x 15840 twips, A4 portrait is 11906 x 16838 twips; swap width/height when orientation is landscape. Build a small helper that returns `{ pageSize: { width, height }, margins: { top, right, bottom, left, header, footer, gutter } }` from the existing inch-based margins.
- Do not change HTML or PDF behavior. The Mermaid rasterization step is only run for DOCX export.
