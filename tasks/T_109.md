# T_109: Replace the ngrok SDK with ngrok CLI integration

## Status

Done

## Summary

Replace the bundled `@ngrok/ngrok` native SDK (added in T_108) with the user's externally-installed ngrok CLI, for minimal bundled dependencies. The tunnel behavior is the same (optional, off-by-default public tunnel to the loopback MCP server, optional custom domain with fallback), but Nexus now spawns the ngrok CLI as a background process and reads the authtoken from the ngrok CLI's own configuration. The ngrok authtoken is no longer stored in Nexus settings.

## Implementation Notes

- Dependencies (`package.json`):
  - Remove `@ngrok/ngrok` from dependencies (`npm uninstall @ngrok/ngrok`).
  - Remove the electron-builder `asarUnpack` entry that was added for the `@ngrok/ngrok` native binary.
- Settings (`src/lib/settings.ts`):
  - Remove `ngrokAuthToken` from `McpServerSettings`, `createDefaultMcpServerSettings`, and `sanitizeMcpServerSettings`.
  - Keep `ngrokEnabled` (boolean, default `false`) and `ngrokDomain` (string, default `""`).
  - Update `src/lib/settings.test.ts`: remove `ngrokAuthToken` from the full-object `createDefaultSettings`, font-only load, and save assertions, and from the dedicated ngrok load/sanitize tests (keep `ngrokEnabled` and `ngrokDomain` assertions).
- Tunnel module (`electron/ngrok-tunnel.cjs`): rewrite to spawn the ngrok CLI instead of the SDK.
  - Spawn with `child_process.spawn` (no shell): `ngrok http <port> [--domain <domain>] --log stdout --log-format json`.
  - Read the agent's JSON log lines from stdout; resolve the public URL from the line whose `msg` is `"started tunnel"` (it carries the `url` field).
  - Detect spawn `ENOENT` as a clear "ngrok CLI not found - install ngrok and run `ngrok config add-authtoken`" error.
  - Treat error-level log lines (`lvl === "error"` or `"crit"`) as failures; map authtoken errors (for example `ERR_NGROK_4018` or text containing `authtoken`) to a "run `ngrok config add-authtoken`" message. Do not treat info lines that contain `err:"<nil>"` as errors.
  - Keep the module interface: `ensureTunnel({ port, domain })`, `stopTunnel()`, `getTunnelState()` returning `{ connected, url, error, domainFallback }`.
  - Track the active port and domain; no-op when unchanged and connected; respawn when either changes; serialize calls with a pending promise.
  - On a custom-domain failure, retry once without the domain (fall back to a random URL and set `domainFallback`).
  - Kill the child on `stopTunnel()` and on app quit; reflect an unexpected child exit as a disconnected tunnel.
  - Use a startup timeout so a hung start cannot block the configure flow. Validate the domain against a hostname pattern before passing it to the CLI.
  - On macOS, prepend the standard Homebrew bin locations (`/opt/homebrew/bin`, `/usr/local/bin`) to `PATH` before spawning (idempotent, darwin-only), because a Finder/Dock-launched app does not inherit the shell `PATH` and would otherwise not find a Homebrew-installed ngrok.
  - Confirmed CLI behavior: the `started tunnel` JSON line carries the public `url`, and the agent reads the authtoken from the ngrok config (`ngrok.yml`).
- Main process (`electron/main.cjs`):
  - In the `mcp:configure` handler, stop forwarding or sanitizing an ngrok authtoken.
  - Gate the tunnel on `result.ok && result.listening && ngrokEnabled`. Call `ensureTunnel({ port, domain })` (domain from the config) when gated on, otherwise `stopTunnel()`.
  - Keep returning `ngrok: { enabled, connected, url, error, domainFallback }`.
  - Keep stopping the tunnel on `will-quit`.
- Preload (`electron/preload.cjs`): no change needed; `configureMcpServer` forwards the whole config object.
- Types (`src/electron.d.ts`): remove `ngrokAuthToken` from `ConfigureMcpServerInput` (keep `ngrokDomain`). `McpNgrokStatus` stays `{ enabled, connected, url, error, domainFallback }`.
- App shell (`src/App.tsx`): remove `ngrokAuthToken` from the `mcp:configure` payload and the effect dependency array.
- Settings dialog (`src/components/settings/SettingsDialog.tsx`):
  - Remove the ngrok authtoken input and its change handler.
  - Add help text that the tunnel requires the ngrok CLI installed and an authtoken configured with `ngrok config add-authtoken <token>`.
  - Keep the custom-domain field, the public URL and public `/mcp` endpoint with copy controls, the tunnel error display, the domain-fallback notice, and the no-authentication warning.
