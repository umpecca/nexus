# T_061: Render Markdown highlights in HTML and PDF exports

**Task Status:** Completed

**Task Description:** Fix export rendering so Markdown highlight syntax appears as highlighted text instead of raw `==highlighted text==` source in generated HTML and PDF files.

## Context

Nexus supports highlighted inline text in the visual editor and demo document, but the export renderer used Marked without a highlight extension. As a result, exported PDF output showed the literal Markdown delimiters.

## Acceptance Criteria

1. `==highlighted text==` exports as highlighted text.
2. The shared HTML export path receives the same highlight rendering as PDF export.
3. Highlighted text can contain nested inline Markdown such as bold or italic.
4. Code spans and fenced code blocks keep literal `==` text unchanged.
5. Existing Mermaid, admonition, image, font-size, page-size, and margin export behavior remains unchanged.

## Implementation Notes

- Added a Marked inline extension for `==...==` highlight syntax in the Electron export pipeline.
- Rendered highlight tokens as `<mark>` elements.
- Added export CSS for highlighted text so PDF output preserves the visual background.

## Verification

Run `npm run test`, `npm run build`, `node --check electron/main.cjs`, `git diff --check`, and a direct Marked parser smoke test for highlights.
