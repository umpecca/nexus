# T_053: Exclude frontmatter from PDF export

**Task Status:** Completed

**Task Description:** Omit leading YAML frontmatter metadata from PDF exports while preserving the rest of the rendered Markdown document.

## Context

Nexus supports frontmatter in the editor and demo document, but PDF exports are intended as reader-facing output. Metadata blocks should not appear in exported PDFs, while the Markdown source and other export behavior should remain unchanged.

## Acceptance Criteria

1. File/Export as PDF omits a leading YAML frontmatter block delimited by `---` and `---` or `...`.
2. PDF export still renders the remaining Markdown content, including Mermaid diagrams, images, tables, code blocks, and admonitions.
3. HTML export behavior remains unchanged.
4. Saving and editing preserve frontmatter in the Markdown document.

## Implementation Notes

- Added a main-process helper that strips only leading YAML-style frontmatter from Markdown before PDF rendering.
- Added an internal `excludeFrontmatter` option to the shared Markdown export renderer.
- Enabled frontmatter exclusion only for the `file:export-pdf` handler.

## Verification

Run `npm run build`, `npm run test`, and `git diff --check`. Manually export the built-in demo document to PDF and confirm the frontmatter metadata is not visible in the output.
