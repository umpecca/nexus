# T_019: Show the active document path in the application title

**Status:** Done

**Task Description:** Update Nexus so the native application title identifies the active edited document like a traditional desktop editor.

## Acceptance Criteria

1. A document without a current file path shows the app name and untitled state in the application title.
2. An opened or saved document shows the app name, file name, and full file path in the application title.
3. File/New resets the title back to the untitled state because it clears the current file path.
4. The title is driven by the same renderer document state used by new, open, save, and save as.

## Implementation Notes

- Added a small title formatter in `src/App.tsx`.
- Added a React effect that updates `document.title` whenever the current file path changes.
- Electron follows the renderer document title for the native window title, so no broader IPC surface was needed.
